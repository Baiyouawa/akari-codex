# 检索并收集可支撑该课程路线的高质量资料与综述来源，优先整理可追溯链接、标题、摘要与适用章节

Generated: 2026-03-26

## Goal Description
围绕既定课程路线，立即重启并行综述调研流程，系统检索、筛选、整理可用于课程设计、内容论证与延伸阅读的高质量资料来源。输出结果应以“可追溯、可复核、可映射到课程章节”为核心，优先收集具有稳定来源链接、明确标题、摘要/核心内容概述、发表信息与章节适配建议的综述类、权威指南类、经典教材/官方文档类资料。最终形成一份结构化资料清单，支持后续课程大纲细化、讲义撰写、案例补充与参考文献建设。

## Acceptance Criteria

- AC-1: 已完成面向课程路线全部核心主题的资料检索与初筛
  - Positive Tests: 至少覆盖课程路线中的每个一级主题；每个主题均有对应资料候选；检索范围包含综述、权威机构文档、经典论文/教材、官方技术文档中的至少两类
  - Negative Tests: 仅覆盖部分章节；只提供零散材料而无体系性；资料类型单一且无法支撑课程设计

- AC-2: 每条入选资料均具备可追溯的基础元数据
  - Positive Tests: 每条资料至少包含标题、作者/机构、年份、来源、可访问链接；链接可回溯到原始发布页、DOI页、出版社页、官方文档页或可信数据库页
  - Negative Tests: 使用无法追溯来源的二次转载；缺失标题或来源信息；链接指向聚合页、失效页或不明页面

- AC-3: 每条资料均附带简要摘要与课程适配说明
  - Positive Tests: 每条资料附 50–150 字摘要或要点概述；明确标注适用章节、适用目的（导论/理论支撑/案例/进阶阅读/方法综述）
  - Negative Tests: 仅罗列标题和链接；摘要过于空泛；未说明适用于课程的哪个章节

- AC-4: 资料质量经过分层筛选并标注优先级
  - Positive Tests: 对资料标注“核心/重要/补充”或同等优先级；优先选择高被引综述、顶级期刊/会议、权威机构发布、主流教材与官方标准文档；说明入选理由
  - Negative Tests: 所有资料混排无优先级；低质量博客、自媒体、无审稿内容与高质量来源混杂；未解释筛选依据

- AC-5: 形成支持并行推进的结构化交付格式
  - Positive Tests: 输出按章节或主题分组；可直接用于后续课程编排；字段统一，便于继续扩充；具备去重结果与初步缺口标记
  - Negative Tests: 资料散乱堆积；字段格式不统一；存在明显重复；无法直接衔接后续课程开发流程

- AC-6: 对检索过程与范围有基本记录，保证可复现
  - Positive Tests: 记录主要检索关键词、数据库/平台、筛选口径与时间范围；可说明为何纳入或排除某类资料
  - Negative Tests: 无检索策略记录；无法解释资料来源；后续无法复现同类检索结果

## Path Boundaries

### Upper Bound (Maximum Scope)
构建一份面向整条课程路线的高质量资料综述库，覆盖全部一级与二级章节。对每个章节至少整理 5–10 条高价值来源，其中包含综述论文、代表性原始论文、权威教材章节、官方标准/白皮书、技术文档与高质量案例。所有条目均含完整元数据、稳定链接、摘要、适用章节、优先级、关键词、来源类型、使用建议与备注。同步记录检索式、数据库、筛选标准、去重逻辑与内容缺口，形成可持续更新的资料台账。

### Lower Bound (Minimum Scope)
快速完成面向课程主干章节的首轮资料清单，每个一级章节至少整理 2–3 条高质量来源，且每条包含标题、链接、摘要与适用章节。优先保证综述类和权威来源齐备，先满足课程搭建所需的基础支撑，再在后续迭代中补全深度资料。

### Allowed Choices
- Can use: 学术数据库（Google Scholar、Crossref、PubMed、IEEE Xplore、ACM Digital Library、Web of Science、Scopus、arXiv）、出版社页面、DOI 链接、大学课程公开资源、权威机构官网、标准组织官网、主流技术官方文档、经典教材目录页与作者主页
- Can use: 章节映射表、资料优先级标签、统一字段模板、并行主题拆分、去重清单、缺口标记、二次复核机制
- Cannot use: 无法确认来源的转载内容、采集型内容农场、自媒体观点文、仅有截图无原始出处的材料、失效且无法回溯的链接、与课程主题相关性极弱的泛资料
- Cannot use: 未标注出处的摘要拼接、仅依赖单一搜索引擎的结果、未经筛选直接堆叠的大批低质参考链接

## Dependencies and Sequence

### Milestones
1. 明确课程路线拆分结构
   - 获取现有课程路线、章节目录、目标受众与课程深度要求
   - 将路线拆分为一级主题、二级主题与重点知识点
   - 为每个主题建立资料需求说明：综述、基础理论、案例、实践文档、扩展阅读

2. 建立检索框架与筛选口径
   - 制定统一字段模板：标题、链接、来源、作者/机构、年份、摘要、适用章节、优先级、备注
   - 确定优先来源池与备用来源池
   - 设定纳入标准：权威性、相关性、可访问性、可追溯性、时效性、综述价值

3. 并行开展主题检索
   - 按章节或主题分组并行推进
   - 为每组主题准备中英文关键词、同义词、领域术语与限定词
   - 优先检索综述论文、survey、tutorial、state-of-the-art、systematic review、handbook、official guide 等类型资料

4. 完成首轮筛选与去重
   - 剔除重复、低相关、不可追溯、低可信来源
   - 保留可直接支撑课程路线的核心条目
   - 对每个章节形成首轮候选列表并标注质量层级

5. 生成结构化资料卡片
   - 为每条资料补齐标题、链接、来源、年份、摘要与章节适配建议
   - 用统一格式整理，确保可直接复制进入课程参考文献库
   - 补充“推荐用途”字段，如导入阅读、主讲支撑、案例补充、进阶阅读

6. 开展二轮复核与缺口补齐
   - 检查章节覆盖度与资料类型平衡度
   - 对空白或薄弱章节追加定向检索
   - 校验链接稳定性与元数据一致性

7. 输出最终资料清单与检索记录
   - 按课程章节输出整理结果
   - 附带检索关键词、主要平台、筛选原则与缺口说明
   - 标记后续可继续扩展的方向，便于并行综述流程持续推进

## Implementation Notes
- 以“章节覆盖完整、来源可追溯、摘要可复用、映射清晰”为整理原则
- 优先顺序建议为：高质量综述 > 权威指南/标准 > 经典教材/教程 > 代表性论文 > 官方实践文档
- 若课程路线尚未完全定稿，先基于现有主干结构建立“临时主题树”，避免等待造成检索停滞
- 并行调研时应统一命名规范与字段模板，避免不同主题组输出风格不一致
- 摘要应以“对课程有什么用”为导向，不仅复述论文摘要，还应提炼其教学价值
- 对同一主题下的多篇相似综述，应优先保留更新、更权威、覆盖更完整者
- 对经典但年份较早的资料，可保留并标注“历史奠基/理论基础”
- 对近年快速变化的主题，应额外补入最近 2–3 年的更新综述或官方文档
- 建议显式标注资料适用层次：入门、进阶、研究导向、实践导向
- 最终结果宜采用表格或统一卡片结构，便于后续排序、筛选与增补
- 代码中不应包含 AC-、Milestone 等计划术语
