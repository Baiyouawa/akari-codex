# 将筛选出的十篇综述尽可能下载到本地，保存PDF或可读文本，并整理准确本地路径清单

Generated: 2026-03-26

## Goal Description
在已筛选出十篇 multi-agent 最新综述论文的前提下，组织执行下载与落地保存工作：尽可能获取每篇论文的本地 PDF 文件；若无法获得 PDF，则保存可读文本版本（如 HTML 转文本、官方页面正文摘录、开放仓库版本等）；最终形成一份准确、可验证、可复用的本地路径清单，确保后续 Agent 集群能够直接基于本地材料开展精读、中文 Markdown 总结与 idea 设计。

## Acceptance Criteria

- AC-1: 十篇目标综述均完成逐篇检索与下载尝试，并记录处理结果
  - Positive Tests: 十篇论文每篇都有唯一标识记录，包含标题、来源链接、检索状态、下载结果、失败原因或替代方案
  - Negative Tests: 仅下载成功部分论文而未记录其余论文的失败原因；出现标题混淆、重复记录或漏项

- AC-2: 对可获取全文的论文，已成功保存本地 PDF 或可读文本
  - Positive Tests: 本地目录中存在可打开的 PDF 文件，或存在结构清晰、可阅读的 txt/md/html 文本文件；文件名与论文标题或编号可对应
  - Negative Tests: 保存的是损坏文件、登录跳转页、反爬错误页、空白文件，或文件内容与目标论文不匹配

- AC-3: 对无法直接下载 PDF 的论文，已按优先级执行替代获取方案
  - Positive Tests: 尝试官方页面、开放获取版本、作者主页、预印本、机构仓库、可读网页正文等，并保存最优可用版本
  - Negative Tests: 遇到付费墙或访问限制后直接放弃，未尝试合法替代来源；保存未经核验的非目标文档

- AC-4: 形成一份准确的本地路径清单，能够支撑后续自动化处理
  - Positive Tests: 路径清单包含论文标题、最终保存格式、绝对路径或项目相对路径、文件存在性验证结果
  - Negative Tests: 路径缺失、路径错误、仅有人类可读描述而无真实文件路径、路径与文件不一致

- AC-5: 文件组织规范统一，便于后续批处理与引用
  - Positive Tests: 统一命名规则、统一目录层级、去重完成、特殊字符已处理；同一论文的主文件与补充文件关系明确
  - Negative Tests: 文件散落多个目录、命名混乱、同一论文多个版本无法区分、中文或特殊字符导致脚本难以处理

- AC-6: 下载过程与结果具备可审计性
  - Positive Tests: 保留来源 URL、访问日期、获取方式、状态说明；对失败项记录限制条件，如付费墙、地区限制、站点故障
  - Negative Tests: 最终只有文件，没有来源信息；后续无法追溯文件来源或判断合法性与版本可靠性

- AC-7: 交付结果能直接服务后续“精读—总结—idea 设计”流程
  - Positive Tests: 每篇论文至少有一份可供程序或人工阅读的本地材料，且路径清单可供下游脚本读取
  - Negative Tests: 材料虽下载但无法解析、扫描件不可读、路径清单格式不稳定导致后续 Agent 无法消费

## Path Boundaries

### Upper Bound (Maximum Scope)
对十篇综述逐篇进行多源合法检索与下载，优先获取正式 PDF；如受限则获取开放获取版本、预印本、作者接受稿、机构仓库版本或高质量网页正文，并统一转换为规范化本地文件。建立标准目录结构、命名规则、元数据表与路径清单，逐项校验文件可打开性、内容匹配性和来源可靠性；对失败项输出明确原因与建议补救动作。最终交付包括：本地文件全集、路径清单、来源清单、失败报告、版本说明，可直接供后续自动摘要、引用抽取和 Markdown 总结流水线使用。

### Lower Bound (Minimum Scope)
完成十篇论文的基本下载尝试；对能获取的论文保存本地 PDF 或可读文本；对无法获取的论文记录失败原因；输出一份最小可用路径清单，至少包含标题、保存格式、文件路径、状态。

### Allowed Choices
- Can use: 浏览器手动下载、命令行下载工具、合法开放获取渠道、机构仓库、作者主页、预印本平台、网页转文本、统一目录管理、校验脚本、CSV/JSON/Markdown 路径清单
- Cannot use: 绕过付费墙的非法手段、未授权账号共享、破解 DRM、保存与目标论文无关内容冒充全文、篡改原文内容、输出不存在的本地路径

## Dependencies and Sequence

### Milestones
1. 明确十篇目标综述的唯一清单，锁定标准标题、年份、作者、DOI 或官方链接，避免后续误下载
2. 设计本地目录结构与命名规则，确定 PDF、文本、元数据、日志的保存位置
3. 为每篇论文建立检索任务卡，填写候选来源优先级：官方出版页、开放获取页、预印本、作者主页、机构仓库、可读网页
4. 执行第一轮下载，优先获取正式 PDF，并立即做文件可打开性与内容匹配检查
5. 对第一轮失败项执行第二轮替代方案获取，保存可读文本或次优合法版本
6. 对已保存文件进行统一重命名、去重、格式归档，并补充来源 URL、访问日期、版本说明
7. 生成本地路径清单，逐项校验路径真实存在、文件可读、标题对应正确
8. 输出最终结果包：本地文件集合、路径清单、失败项说明，为后续精读与总结 Agent 提供输入

## Implementation Notes
- 先做“目标论文去歧义”，避免同名综述、早期版本、会议扩展版混淆
- 推荐目录结构示例：`papers/reviews/pdf/`、`papers/reviews/text/`、`papers/reviews/meta/`、`papers/reviews/logs/`
- 推荐命名格式示例：`[序号]_[年份]_[第一作者姓]_[标题短名].[pdf|md|txt|html]`
- 若标题过长，保留稳定短名，并在元数据文件中保留完整标题
- 对每篇论文至少保存以下字段：标题、作者、年份、DOI、来源 URL、获取方式、文件格式、本地路径、状态、备注
- PDF 校验至少包括：文件可打开、页数大于零、首页标题或正文与目标论文一致
- 文本版校验至少包括：正文长度达标、非错误页、含目标标题或核心摘要信息
- 同时存在多个版本时，明确“主版本”和“备选版本”，避免后续摘要时误用
- 对付费墙论文，优先记录官方页并寻找合法开放版本，不应仅保存摘要页充当全文
- 路径清单建议同时输出机器可读和人工可读两种形式，如 `paths.csv` 与 `paths.md`
- 建议在路径清单中增加 `exists=true/false`、`readable=true/false` 字段，便于后续脚本检查
- 若网页正文转文本，需保留原始来源链接与抓取时间，避免后续引用失真
- 应对下载失败的常见原因进行标准化标注，如：`paywalled`、`not_found`、`blocked`、`metadata_only`
- 可在最终交付中增加“建议后续动作”列，例如联系作者、通过机构订阅获取、等待开放版本
- 代码中不应包含 AC-、Milestone 等计划术语
