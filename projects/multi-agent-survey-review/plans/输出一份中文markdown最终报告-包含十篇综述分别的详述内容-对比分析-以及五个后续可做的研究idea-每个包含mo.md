## Path Boundaries

## Dependencies and Sequence

### Milestones
1. 项目启动与任务框架冻结
   - 明确最终交付物边界：十篇综述详述、对比分析、五个 idea、中文 Markdown 形式。
   - 确定 Agent 角色分工与通信机制。
   - 制定统一模板：文献筛选表、阅读笔记表、综述详述模板、对比分析模板、idea 模板。

2. 文献检索与候选池建立
   - 检索 multi-agent 相关最新综述，包括传统多智能体系统、多智能体强化学习、LLM-based multi-agent systems、agent communication、coordination、planning、benchmarking 等方向。
   - 汇总候选综述长名单，并记录题名、年份、平台、摘要、链接、是否可下载。
   - 初步去重与主题归类。

3. 十篇核心综述筛选定稿
   - 基于代表性、时效性、主题覆盖度、引用价值、综述质量筛选最终十篇。
   - 保证选题之间既有共性也有差异，避免十篇过于集中在单一子领域。
   - 输出“入选理由 + 淘汰理由”的简要说明，便于后续追溯。

4. 原文获取与精读分发
   - 下载或打开十篇综述原文。
   - 按主题分发给阅读 Agent，并使用统一模板进行精读。
   - 对重点章节进行深度提炼，包括问题定义、分类法、评价框架、未来方向。

5. 单篇综述结构化总结
   - 将阅读笔记转写为中文详述内容。
   - 保持十篇小节结构一致，确保后续横向比较可对齐。
   - 提炼每篇综述的独特贡献与局限，避免只做复述。

6. 跨综述对比分析
   - 建立比较维度：主题范围、分类方式、任务对象、方法演进、数据与评测、应用领域、挑战与未来方向。
   - 形成表格、矩阵或分组文字总结。
   - 识别共识、分歧、断层与机会点。

7. 后续研究 idea 设计
   - 基于对比分析提炼空白问题或未充分连接的方向。
   - 设计五个研究 idea，每个完整包含 motivation、问题定义、方法设计、预期贡献、潜在风险。
   - 检查 idea 与综述分析之间的逻辑链是否充分。

8. 报告整合与中文 Markdown 成稿
   - 将引言、十篇详述、对比分析、五个 idea、总结建议整合为单文档。
   - 统一术语、风格、标题层级与 Markdown 规范。
   - 确保可直接交付、阅读和后续扩展。

9. 质量复核与最终验收
   - 对事实性信息做交叉核查：论文题名、作者、年份、核心结论。
   - 对结构完整性进行检查：十篇是否齐全、对比分析是否成型、五个 idea 是否五要素完整。
   - 对语言流畅性、逻辑性与格式进行终审后输出定稿。

- 建议采用“协调 Agent + 检索 Agent + 筛选 Agent + 阅读 Agent + 对比分析 Agent + 创意设计 Agent + 审校 Agent”的协作方式。

- 综述筛选标准建议至少包含：是否为综述性论文、是否直接涉及 multi-agent、是否具有清晰分类框架、是否具备代表性、是否可获得原文。
- 单篇综述详述建议统一模板：论文信息、研究背景、综述目标、内容结构、方法分类、关键观点、优点、局限、适用启示。
- 对比分析建议优先采用表格化呈现，再配合文字解释，以提升报告可读性和结论密度。
- 研究 idea 不宜脱离已有综述结论，应尽量从“现有方法未统一评价”“真实环境协作不足”“多 Agent 安全与对齐不足”“通信成本与可解释性冲突”“长期任务与开放世界泛化”等问题切入。

- 对存在争议或不确定的信息，优先回查原文而非依赖摘要页或第三方解读。
- 若某些综述内容高度重叠，可在单篇详述中保留个体特色，在对比分析中突出其差异化价值。
- 最终 Markdown 报告应避免“流水账式”堆砌，重点体现筛选逻辑、知识结构与研究洞见。
