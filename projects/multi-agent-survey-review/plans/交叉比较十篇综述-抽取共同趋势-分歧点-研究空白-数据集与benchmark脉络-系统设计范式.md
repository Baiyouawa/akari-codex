# 交叉比较十篇综述，抽取共同趋势、分歧点、研究空白、数据集与benchmark脉络、系统设计范式

Generated: 2026-03-26

## Goal Description
围绕“multi-agent 最新综述调研”这一目标，组织可并行协作的 Agent 集群，对不少于十篇高相关综述论文进行系统化获取、筛选、下载、精读、信息抽取与交叉比较，最终形成一份中文 Markdown 研究总结与后续 idea 设计底稿。  
任务产出应覆盖以下核心维度：

1. **共同趋势**：识别十篇综述在研究主题、技术演进、任务设定、评测趋势上的一致性结论。
2. **分歧点**：识别不同综述在问题定义、方法分类、能力边界、评测标准、未来方向上的不一致之处。
3. **研究空白**：总结现有综述尚未充分讨论或仍缺乏共识的议题，形成可用于后续选题的空白地图。
4. **数据集与 benchmark 脉络**：梳理各综述提及的数据集、基准任务、评测协议、时间演进关系与适用场景。
5. **系统设计范式**：抽取 multi-agent 系统在角色分工、通信机制、记忆设计、工具使用、规划执行、反馈闭环、安全约束等方面的典型架构模式。
6. **可执行后续工作基础**：为后续 idea 设计提供结构化证据库，包括论文元数据、摘要级结论、细粒度标签、对比矩阵、引用映射和候选研究问题。

该计划不仅关注“读懂十篇综述”，还要求建立一套可复用的综述交叉分析流程，使 Agent 集群能够高效协作、降低重复劳动、提高结论可追溯性，并保证中文 Markdown 总结具备研究汇报与后续立项的双重可用性。

## Acceptance Criteria

- AC-1: 完成不少于十篇与 multi-agent 主题高度相关的综述论文样本集构建，并附带可追溯的基础元数据
  - Positive Tests: 收集到 10 篇及以上综述；每篇具备标题、作者、年份、来源、链接、摘要、主题标签、纳入理由；样本时间分布与主题覆盖基本合理
  - Negative Tests: 仅收集零散论文而无综述属性；不足 10 篇；缺少来源链接与纳入依据；存在明显与 multi-agent 主题不相关的论文混入

- AC-2: 建立统一的信息抽取模板，并完成对十篇综述的逐篇结构化精读记录
  - Positive Tests: 每篇综述均按统一模板记录研究对象、分类体系、关键观点、数据集、benchmark、系统设计要素、局限性、未来方向；抽取字段格式一致
  - Negative Tests: 各篇记录格式随意且不可比；仅有摘要翻译无精读信息；缺少关键维度导致后续无法交叉比较

- AC-3: 输出跨论文对比矩阵，能够清晰识别共同趋势与主要分歧点
  - Positive Tests: 至少形成一份“论文 × 维度”的对比表；能列出多个跨综述共识结论与多个分歧议题；每条结论可回溯到具体论文证据
  - Negative Tests: 只有单篇摘要堆叠，没有横向比较；结论无法对应来源；分歧点仅凭主观判断而无支撑

- AC-4: 系统梳理数据集与 benchmark 脉络，形成按任务、时间或场景组织的结构化清单
  - Positive Tests: 能归纳多篇综述共同提及的数据集/benchmark；说明其用途、评测维度、适用任务与演进关系；指出哪些 benchmark 被过度使用或缺乏统一标准
  - Negative Tests: 只罗列名字无上下文；无法区分数据集与 benchmark；没有时间脉络与任务归类

- AC-5: 提炼 multi-agent 系统设计范式，并形成可复用的架构分类框架
  - Positive Tests: 至少覆盖角色分工、通信机制、协调策略、记忆机制、工具调用、规划反馈、安全治理等多个维度；能总结出若干典型范式并举证
  - Negative Tests: 仅泛泛描述“多智能体协作”；无清晰分类；无法映射回综述原文观点

- AC-6: 识别研究空白与未来机会，形成后续 idea 设计输入
  - Positive Tests: 给出若干研究空白，说明其证据来源、为什么重要、当前缺口何在、可行切入方向；形成候选 idea 列表
  - Negative Tests: 只写“未来值得研究”之类空话；没有证据链；空白点与综述内容脱节

- AC-7: 交付一份中文 Markdown 总结，结构完整、可读性强、支持后续直接扩展为汇报或立项文档
  - Positive Tests: Markdown 包含背景、样本说明、单篇摘要、交叉比较、趋势、分歧、空白、benchmark 脉络、系统范式、idea 建议、参考链接；目录与标题层级清晰
  - Negative Tests: 输出为零散笔记；无统一结构；中英文混杂严重；内容不可直接用于团队协作

- AC-8: Agent 集群分工明确，任务流程具备并行性与质量复核机制
  - Positive Tests: 明确检索、下载、去重、精读、抽取、对比、审校、汇总等角色；存在交叉复核或抽样校验流程；能追踪任务状态
  - Negative Tests: 所有工作混在单线程流程中；无角色划分；无复核导致信息遗漏或不一致

## Path Boundaries

### Upper Bound (Maximum Scope)
最完整方案应包括：

- 检索并筛选 **10–15 篇** multi-agent 相关高质量综述，覆盖 LLM-based multi-agent、传统 MAS、agent collaboration、agent planning、tool use、evaluation、safety/governance 等子方向
- 使用 Agent 集群并行完成：
  - 文献检索与候选池构建
  - PDF/网页版本下载与归档
  - 元数据清洗与去重
  - 逐篇精读与结构化抽取
  - 引文链补充与时间线构建
  - benchmark / dataset 专题梳理
  - system design pattern 专题梳理
  - 研究空白与 idea 生成
  - 中文 Markdown 主报告撰写与交叉审校
- 为每篇综述生成：
  - 200–500 字中文摘要
  - 核心贡献点
  - 分类体系提炼
  - 涉及数据集与 benchmark 记录
  - 方法/系统范式标签
  - 局限性与偏见说明
- 形成多层次交付物：
  - 主报告 Markdown
  - 论文信息表（CSV/JSON/Markdown 表格均可）
  - 交叉对比矩阵
  - benchmark 时间线
  - 系统设计范式图谱
  - 候选 idea 清单
- 对每项结论保留证据引用，确保可追溯
- 对结论进行至少一轮二次复核，减少误读与幻觉性归纳

### Lower Bound (Minimum Scope)
最简可行方案应包括：

- 至少筛选并精读 **10 篇** multi-agent 综述
- 每篇至少记录：标题、年份、来源、摘要、主要主题、关键结论、未来方向
- 产出一份中文 Markdown，总结：
  - 共同趋势
  - 分歧点
  - 研究空白
  - 数据集/benchmark 清单
  - 系统设计范式初步分类
- 至少提供一张交叉对比表
- 至少给出 3 个后续研究 idea
- 所有结论需能对应到具体论文，不允许纯主观臆测

### Allowed Choices
- Can use:
  - 学术搜索引擎、论文数据库、开放获取平台、作者主页、机构仓库
  - Agent 并行协作方式进行检索、下载、摘要、抽取、对比、审校
  - 表格、知识卡片、标签体系、引用映射、时间线等结构化方法
  - 中文撰写，必要时保留英文术语
  - Markdown 作为主交付格式
- Cannot use:
  - 非综述类论文替代综述样本数量要求
  - 仅依据二手博客、论坛帖子、社交媒体内容做核心结论
  - 无来源、无链接、无证据链的总结
  - 将单篇论文摘要机械拼接冒充交叉比较
  - 用未经核对的自动生成内容直接作为最终结论
  - 忽略论文时间、主题差异而强行合并结论

## Dependencies and Sequence

### Milestones
1. **任务定义与范围冻结**
   - 明确 multi-agent 主题边界、时间范围、综述纳入标准、输出结构
   - 确定“十篇综述”是否优先覆盖最新 2–3 年，并保留经典综述作对照
   - 设定统一命名、存储、引用和标签规范

2. **Agent 集群组织与角色分配**
   - 设置检索 Agent、筛选 Agent、下载 Agent、精读 Agent、抽取 Agent、对比 Agent、审校 Agent、汇总 Agent
   - 明确输入输出接口，避免重复劳动
   - 建立任务看板与状态字段，如候选中、已下载、待精读、已抽取、待复核、已汇总

3. **文献检索策略制定**
   - 设计关键词组合：multi-agent、LLM agents、agent society、agent collaboration、survey、review、benchmark、evaluation、framework 等
   - 在多个来源检索候选综述，形成初始候选池
   - 记录检索式、来源平台、时间戳，保证可复现

4. **候选综述筛选与去重**
   - 根据纳入标准筛选候选论文：综述属性、主题相关性、来源质量、可获取性、时效性
   - 去除重复版本、低相关论文、非正式稿件
   - 形成最终十篇核心样本和若干备选样本

5. **文献下载与资料归档**
   - 下载 PDF、网页快照或可访问版本
   - 统一存储目录，建立文件名规范
   - 保存题录信息与访问链接，避免后续失链

6. **统一抽取模板设计**
   - 定义每篇综述的必抽字段：
     - 基础信息
     - 研究问题定义
     - 分类体系
     - 核心趋势
     - 争议/分歧
     - 数据集与 benchmark
     - 系统设计范式
     - 局限性
     - 未来方向
   - 先对 1–2 篇样本试填，修订模板后再批量展开

7. **逐篇精读与结构化抽取**
   - 各精读 Agent 按模板完成全文阅读与信息记录
   - 对难点段落、分类定义、表格信息进行重点提取
   - 标记直接证据位置，便于复核与引用

8. **质量复核与一致性校验**
   - 抽样复读关键论文，检查抽取内容是否失真
   - 对标签命名、分类口径、benchmark 名称做统一清洗
   - 纠正跨论文对比中可能出现的同名异义或异名同义问题

9. **交叉比较分析**
   - 构建“论文 × 分析维度”矩阵
   - 汇总共同趋势，如研究热点、架构收敛方向、评测偏好
   - 汇总分歧点，如 agent 定义边界、协作机制分类、评测有效性判断
   - 提炼高频共识与低频但关键的争议议题

10. **数据集与 benchmark 脉络梳理**
    - 建立数据集/benchmark 清单
    - 标注其所属任务、输入输出形态、评测目标、使用频率、局限性
    - 如可能，绘制时间线或代际演化关系

11. **系统设计范式抽取**
    - 从综述中归纳系统架构模式，如：
      - centralized planner + specialized executors
      - peer-to-peer collaboration
      - hierarchical agents
      - memory-augmented agent society
      - tool-centric agent workflow
      - critic-reflection loop
    - 比较不同范式的适用场景、优缺点、依赖条件与风险点

12. **研究空白与 idea 生成**
    - 基于共识不足、benchmark 缺口、系统瓶颈、评测失配、安全治理不足等角度识别空白
    - 输出可执行的后续 idea，包括问题定义、潜在方法、可能评测方案与创新点假设

13. **中文 Markdown 总报告撰写**
    - 组织内容结构，确保从样本说明到结论建议逻辑完整
    - 插入表格、列表、专题小结、引用链接
    - 保证语言统一、术语一致、结论清晰

14. **最终审校与交付**
    - 检查是否满足十篇综述要求
    - 检查每条关键结论是否可追溯
    - 检查 Markdown 渲染效果、标题结构、链接可用性
    - 输出最终版本与可扩展素材包

## Implementation Notes
- 建议先冻结“什么算综述、什么算 multi-agent 相关”的判定标准，再启动大规模并行检索，避免返工
- 应优先采用“统一抽取模板 + 统一标签字典”，否则后续交叉比较会因字段不一致而失效
- 对“最新综述”与“奠基性综述”应适度平衡：如果全部选最新论文，可能缺乏长期脉络；如果经典论文过多，又会偏离当前趋势
- 对综述的比较应区分“研究对象差异”与“观点分歧”，避免把不同子领域的天然差异误判为学术冲突
- 对 benchmark 的梳理要特别注意：
  - 名称是否规范
  - 是否为真实公开数据集
  - 是否是任务集合、评测协议或 leaderboard，而非传统数据集
  - 是否主要面向单 agent、multi-agent 或通用 agent 框架
- 对系统设计范式的归纳建议采用固定维度：
  - 角色组织
  - 协作拓扑
  - 通信形式
  - 规划与控制
  - 记忆机制
  - 工具与环境交互
  - 反馈与自我修正
  - 安全/治理约束
- 对研究空白的总结应区分：
  - 被多篇综述反复提及的“共识性空白”
  - 仅少量综述指出但具有高潜力的“前沿性空白”
  - 由于 benchmark 缺失导致难以推进的“评测性空白”
- 生成中文 Markdown 时，建议采用以下结构：
  - 项目背景
  - 样本选择标准
  - 十篇综述概览表
  - 单篇精读摘要
  - 交叉比较总表
  - 共同趋势
  - 分歧点
  - 数据集与 benchmark 脉络
  - 系统设计范式
  - 研究空白
  - 后续 idea
  - 参考资料
- 若时间紧张，可先完成“十篇综述概览表 + 交叉比较主表 + 初版总结”，再补全单篇精读细节
- 为便于后续扩展为研究选题，建议每个研究空白后附带“可研究问题”“潜在方法切口”“可行评测路径”
- 输出内容中不应包含 AC-、Milestone 等计划术语
