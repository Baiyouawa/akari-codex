# 对十篇综述逐篇详读，提炼每篇的研究问题、分类框架、核心观点、方法谱系、评测设定、优缺点、局限、值得精读章节与关键图表

Generated: 2026-03-26

## Goal Description
围绕 multi-agent 方向最新且代表性的十篇综述，组织一套可并行执行的调研与精读流程，完成从论文搜集、版本确认、全文下载、逐篇细读、信息抽取、交叉对比，到中文 Markdown 总结与后续研究 idea 孵化的完整实施。最终交付物应支持以下目标：

1. 对每篇综述形成统一结构的阅读档案，至少覆盖：
   - 研究问题
   - 分类框架
   - 核心观点
   - 方法谱系
   - 评测设定
   - 优点与贡献
   - 缺点与局限
   - 值得精读的章节
   - 关键图表及其用途
2. 在十篇综述之间建立横向比较，识别：
   - 共识与分歧
   - 分类口径差异
   - 评测缺口
   - 方法演化主线
   - 未来机会点
3. 输出可复用的中文 Markdown 资料，便于后续：
   - 向团队同步
   - 继续扩展文献池
   - 设计研究 idea
   - 支撑开题、综述写作或技术路线规划
4. 整体流程需适配 Agent 集群并行执行，保证任务拆解清晰、结果格式统一、可审查可回溯。

## Acceptance Criteria

- AC-1: 完成十篇 multi-agent 综述的论文清单确认与资料归档
  - Positive Tests: 十篇论文均有唯一标识信息（标题、作者、年份、来源、链接）；均已下载到本地或知识库；版本明确（arXiv/会议/期刊）；无重复综述混入
  - Negative Tests: 仅记录论文标题但无链接；下载文件不全；同一篇不同版本重复计数；混入非综述论文或与 multi-agent 主题弱相关论文

- AC-2: 每篇综述均形成统一模板的精读笔记
  - Positive Tests: 每篇笔记均包含研究问题、分类框架、核心观点、方法谱系、评测设定、优缺点、局限、值得精读章节、关键图表九个核心字段；字段内容非空且基于正文
  - Negative Tests: 仅有摘要级概述；缺少多个核心字段；内容为泛泛总结且无法对应原文；关键图表未标注来源位置

- AC-3: 对每篇综述的“方法谱系”与“分类框架”给出结构化表达
  - Positive Tests: 能明确说明作者如何划分类别、分类依据是什么、各类方法之间的关系；必要时以层级列表或表格呈现
  - Negative Tests: 只罗列若干名词，没有分类逻辑；将作者原文分类与阅读者自定义分类混淆；无法说明方法演化路径

- AC-4: 对每篇综述的“评测设定”提炼到可比较粒度
  - Positive Tests: 明确总结常见任务、数据集、环境、评价指标、实验协议或 benchmark；指出该综述是否系统讨论评测
  - Negative Tests: 只写“有实验评测”；未区分任务设定与评价指标；未说明综述对评测覆盖是否充分

- AC-5: 每篇综述的优点、缺点与局限分析具备批判性
  - Positive Tests: 能区分“作者贡献”“方法覆盖边界”“叙述偏向”“时效性不足”“缺少统一评测”等不同层面的优缺点与局限
  - Negative Tests: 仅写“内容全面”“值得阅读”等空泛评价；优缺点与局限互相重复；完全脱离原文证据

- AC-6: 明确标注每篇综述中最值得精读的章节与关键图表
  - Positive Tests: 每篇至少给出 2 个建议精读章节和 2 个关键图表/表格，并说明其价值；可定位到章节名或编号
  - Negative Tests: 不指出具体章节编号或标题；只说“方法部分值得读”；关键图表未说明信息密度与用途

- AC-7: 完成十篇综述的横向对比总结
  - Positive Tests: 形成至少一份总览表，比较研究问题、分类维度、方法覆盖、评测覆盖、时间范围、适用读者；能指出互补关系与冲突口径
  - Negative Tests: 只有十篇独立笔记，没有任何跨文献比较；无法总结共同趋势与研究空白

- AC-8: 输出中文 Markdown 交付件可直接用于团队协作
  - Positive Tests: 目录清晰；命名统一；篇目间格式一致；支持复制到知识库；图表引用、原文链接、术语中英文对应完整
  - Negative Tests: 输出为零散片段；格式不统一；缺少标题层级；无法复用到后续报告或 idea 文档

- AC-9: 形成后续 idea 设计的输入材料
  - Positive Tests: 基于十篇综述提炼出若干值得继续追踪的研究空白、方法组合机会或评测改进点；能映射到具体综述证据
  - Negative Tests: 没有任何 future work 汇总；idea 与阅读结果脱节；无法追溯 idea 来源

- AC-10: 整个任务过程可并行、可复核、可追踪
  - Positive Tests: 有明确角色分工、任务顺序、命名规范、审校机制与版本管理；任何一篇综述都能追溯到原始 PDF 与摘要记录
  - Negative Tests: 并行结果格式混乱；审校标准不统一；无法确认某条结论来自哪篇综述或哪个章节

## Path Boundaries

### Upper Bound (Maximum Scope)
最完整方案包括以下内容：

1. 文献层面
   - 筛选并确认十篇最新且高相关的 multi-agent 综述
   - 补充其引用关系、时间线、主题覆盖图谱
   - 对存在版本差异的综述进行版本比对
2. 阅读层面
   - 对十篇全文逐篇精读，而非仅看摘要和结论
   - 重点阅读引言、taxonomy/framework、methods、evaluation、open challenges、conclusion 等章节
   - 抽取所有高价值图表并附解释
3. 结构化产出层面
   - 为每篇输出 1 份详细 Markdown 卡片
   - 为全部十篇输出 1 份横向总表
   - 为整个方向输出 1 份趋势总结与 idea 清单
4. 协作层面
   - 使用 Agent 集群并行：搜集、下载、信息抽取、精读、审校、总整合分工执行
   - 设置统一模板、术语表、引用规范、证据链标注规范
5. 质量层面
   - 对每篇综述至少完成一次交叉复核
   - 对关键结论补充页码、章节号或图表编号
   - 对易混概念进行术语统一
6. 延展层面
   - 从综述延伸出代表性原始论文线索
   - 形成后续深入阅读和研究切入点建议

### Lower Bound (Minimum Scope)
最简可行方案包括以下内容：

1. 确认十篇综述并完成 PDF 下载与基础元信息登记
2. 每篇综述至少输出一份简洁但完整的中文 Markdown 笔记，覆盖九个核心字段
3. 每篇至少指出：
   - 1 个研究问题
   - 1 套分类框架
   - 3 条核心观点
   - 1 组评测信息
   - 2 条优点/缺点或局限
   - 1 个精读章节
   - 1 个关键图表
4. 输出一份十篇对比总表
5. 输出一页后续 idea 草案列表

### Allowed Choices
- Can use:
  - Agent 集群进行并行检索、下载、解析、抽取、审校与整合
  - 学术搜索平台、出版社页面、arXiv、Google Scholar、Semantic Scholar、OpenAlex、Crossref 等进行元信息确认
  - PDF 解析工具、文献管理工具、Markdown 模板、表格工具
  - 中文总结，但保留关键术语英文原文
  - 统一字段模板、标签体系、打分规则、审校清单
  - 引用原文中的图表编号、章节编号、页码信息
- Cannot use:
  - 仅凭摘要、引用摘要或二手博客替代全文阅读
  - 未经核验地编造论文结论、图表含义或评测设置
  - 将非综述论文、教程、观点文章混充为综述
  - 输出无统一结构的随意笔记
  - 省略来源定位信息，导致无法回溯
  - 使用无法复现的私有结论而不保留证据链

## Dependencies and Sequence

### Milestones
1. 明确任务口径与范围
   - 确认“multi-agent”覆盖边界：是否包含 MAS、MARL、LLM-based multi-agent、协作智能体系统、博弈与协调等子方向
   - 确认“最新综述”时间范围与优先级规则
   - 确定十篇综述的入选标准：代表性、时效性、被引、覆盖互补性、可获取性
   - 产出统一阅读模板与命名规范

2. 建立候选综述池并完成十篇定稿
   - 检索候选综述论文，形成不少于 20 篇候选池
   - 去重、筛选、打标签：主题、年份、来源、类型、是否高相关
   - 选出最终十篇，并记录未入选原因
   - 确认每篇的正式版本与下载链接

3. 资料下载与文档归档
   - 下载 PDF、补充 BibTeX/DOI/摘要/作者信息
   - 建立目录结构：原文、提取文本、笔记、图表、总表
   - 为每篇分配唯一编号与文件名
   - 检查文件完整性与可读性

4. 设计 Agent 分工与并行策略
   - 文献检索 Agent：负责候选池与版本确认
   - 下载归档 Agent：负责文件收集与命名统一
   - 精读抽取 Agent：负责按模板填写单篇笔记
   - 图表提取 Agent：负责定位关键图表与表格
   - 审校 Agent：负责字段完整性、证据链与一致性检查
   - 总结整合 Agent：负责十篇横向对比与 idea 生成
   - 设定交接格式，避免并行结果无法合并

5. 执行单篇精读与结构化抽取
   - 按统一模板阅读每篇综述
   - 优先阅读：引言、taxonomy/framework、methods、evaluation、discussion/challenges、conclusion
   - 提炼研究问题、分类框架、核心观点、方法谱系
   - 提取评测设定、优缺点、局限
   - 标记值得精读章节与关键图表
   - 对关键信息标注原文证据位置

6. 完成单篇笔记初稿
   - 每篇整理为中文 Markdown 文档
   - 保留关键术语英文对照
   - 对分类框架和方法谱系采用列表、表格或层级结构呈现
   - 对关键图表说明“图表讲了什么、为什么重要、适合何种读者重点看”

7. 交叉复核与质量审校
   - 检查每篇是否覆盖九个核心字段
   - 检查结论是否与原文一致
   - 检查术语统一、格式统一、图表编号统一
   - 检查评测设定是否真正可比较
   - 对模糊处回到原文二次确认

8. 进行十篇横向比较
   - 构建总表：研究问题、分类依据、方法覆盖、评测覆盖、时间范围、优缺点、局限、推荐人群
   - 识别共同主题与差异主题
   - 总结方法发展主线、评测缺口、综述盲区与冲突口径
   - 找出相互补充的综述组合

9. 形成后续 idea 设计输入
   - 从十篇综述中汇总 open challenges
   - 识别可转化为研究课题的空白点
   - 产出若干 idea 方向，如：
     - 新分类框架整合
     - 统一 benchmark 建议
     - 多子方向综述融合
     - LLM-based multi-agent 与传统 MARL/MAS 的桥接
   - 为每个 idea 附上其来源综述依据

10. 最终交付与归档
   - 输出十篇单篇中文 Markdown 精读笔记
   - 输出一份总览对比 Markdown
   - 输出一份 idea 草案 Markdown
   - 检查目录、链接、文件命名与可读性
   - 准备后续扩展到更多综述或原始论文的接口

## Implementation Notes
- 先统一“综述入选标准”和“multi-agent 子方向边界”，再启动大规模并行，否则后续结果会失配
- 单篇笔记建议采用固定模板，字段顺序保持一致，方便横向合并
- “研究问题”应回答该综述试图整理什么问题域、解决什么阅读痛点，而非泛泛重述 multi-agent 很重要
- “分类框架”需忠实于作者原文；如需补充阅读者自己的重组视角，应单独标注，避免混淆
- “方法谱系”建议从问题设定、交互机制、训练范式、协作/竞争关系、通信机制、规划与工具使用等维度展开
- “评测设定”不要只写 benchmark 名称，应补充任务类型、环境、评价指标和实验比较方式
- “优点、缺点、局限”可分为内容覆盖、理论深度、组织结构、评测梳理、时效性、跨领域视角等不同层次
- “值得精读章节”优先标注信息密度高、定义清晰、分类最完整或 open challenges 最有启发的部分
- “关键图表”优先选择 taxonomy 图、时间线图、方法对比表、benchmark 汇总表、open challenges 总结图
- 对每篇综述应保留最少一条可回溯证据，如章节标题、图号、表号或页码
- 横向比较时应注意不同综述的口径差异，例如有的按应用分，有的按机制分，有的按训练范式分
- 如果十篇综述跨越多个子方向，建议先按子簇分组再汇总，例如传统 MARL、MAS 协同系统、LLM-based multi-agent 等
- idea 设计应从综述明确指出的空白、冲突、未统一的评测和新兴交叉方向中生成，不应脱离证据
- 最终 Markdown 应保持中文主述、英文术语补充、结构清晰、可直接纳入团队知识库
- 代码中不应包含 AC-、Milestone 等计划术语
