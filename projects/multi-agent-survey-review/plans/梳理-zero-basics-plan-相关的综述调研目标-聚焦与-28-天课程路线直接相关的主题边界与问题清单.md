# 梳理 zero-basics-plan 相关的综述调研目标，聚焦与 28 天课程路线直接相关的主题边界与问题清单

Generated: 2026-03-26

## Goal Description
围绕 zero-basics-plan 启动并行综述调研，目标是尽快形成一份可执行、可收敛、直接服务于“28 天课程路线”设计的调研目标定义文档。该文档需要明确以下内容：

1. zero-basics-plan 的综述调研对象是什么，哪些主题与 28 天课程路线直接相关。
2. 调研范围的主题边界：哪些内容必须纳入，哪些内容应排除或延后。
3. 面向课程设计需要回答的核心问题清单，包括学习者画像、先修假设、知识模块、节奏安排、产出形式、难度梯度、实践任务与评估方式等。
4. 并行调研流程的拆分方式，使后续可按主题分支同步推进，并在后续汇总时保持口径一致。
5. 输出结果应能直接支撑下一步的资料检索、综述撰写、课程路线对齐与任务分派。

## Acceptance Criteria

- AC-1: 产出一份明确指向 28 天课程路线的综述调研目标说明
  - Positive Tests: 文档中清楚说明调研是为 28 天课程路线服务，而非泛化为“零基础学习”的宽泛研究；目标描述包含课程设计、主题筛选、问题沉淀三个用途
  - Negative Tests: 目标描述停留在“收集资料”“做调研”；未说明与 28 天课程路线的关系；无法看出输出要服务哪个后续环节

- AC-2: 明确 zero-basics-plan 的主题边界
  - Positive Tests: 给出应纳入的核心主题、可选扩展主题、明确排除主题；边界能区分“课程直接需要”与“背景性但非当前必须”
  - Negative Tests: 主题边界过宽，导致任何入门学习内容都被纳入；或边界过窄，遗漏课程路线所必需的模块

- AC-3: 形成结构化的问题清单，能够指导后续并行调研
  - Positive Tests: 问题清单按类别组织，如用户对象、课程目标、知识结构、时间分配、实践项目、评估标准、风险点；问题可直接转化为检索与综述任务
  - Negative Tests: 问题清单零散重复；问题过于抽象无法执行；没有优先级或分类，无法支持并行拆分

- AC-4: 定义并行综述调研的拆分维度与统一口径
  - Positive Tests: 文档中说明可以按主题模块、课程阶段或问题类别拆分，并定义统一输出字段，如研究问题、关键结论、证据来源、与 28 天路线的关联、待确认事项
  - Negative Tests: 仅提出“并行推进”但没有拆分方式；不同分支调研结果无法合并；缺少统一记录结构

- AC-5: 输出具备最小可执行性，能立即进入资料收集与综述阶段
  - Positive Tests: 文档完成后，执行者可以据此立即开始检索资料、分配子任务、建立综述模板；至少能支撑首轮并行调研启动
  - Negative Tests: 文档仍停留在概念讨论层；缺少执行入口；无法据此分工或建立调研看板

- AC-6: 结果对“零基础”条件有清晰限定，而不是默认无限制降门槛
  - Positive Tests: 明确“零基础”是指哪些能力缺失、哪些基础仍默认具备、哪些前置操作可在课程内补齐；避免把所有基础教育问题都卷入当前范围
  - Negative Tests: 将“零基础”理解为完全无任何数字素养；或完全不定义，导致调研对象失真、课程难度无法设定

## Path Boundaries

### Upper Bound (Maximum Scope)
最完整方案应包括：

- 一份面向 28 天课程路线的综述调研任务定义说明
- 一份主题地图，至少区分：
  - 必须调研主题
  - 与课程强相关但可延后主题
  - 当前明确排除主题
- 一份结构化问题清单，覆盖：
  - 学习者画像与零基础定义
  - 课程目标与最终产出
  - 28 天节奏拆分逻辑
  - 每周能力跃迁目标
  - 知识模块与依赖关系
  - 实践任务与案例形式
  - 工具与环境要求
  - 评估、反馈与完成标准
  - 常见风险与掉队点
- 一份并行调研拆分方案，明确：
  - 可并行的主题包
  - 每个主题包的输入输出
  - 统一的综述记录模板
  - 汇总与冲突消解机制
- 一份优先级定义，区分：
  - 立即需要回答的问题
  - 可在首轮综述后补充的问题
  - 暂不处理的问题
- 一份交付格式建议，使后续可以直接沉淀为调研文档、课程设计文档或任务看板条目

### Lower Bound (Minimum Scope)
最简可行方案应至少包括：

- 一段清晰的调研目标描述，明确其服务于 28 天课程路线
- 一份不超过三级结构的主题边界列表，说明纳入与排除范围
- 一份按类别整理的核心问题清单，至少覆盖学习者、课程结构、实践内容、评估方式四类
- 一种可立即启动的并行调研拆分方式
- 一套统一记录字段，确保不同调研分支可以合并

### Allowed Choices
- Can use: 课程设计视角、综述调研视角、问题树、主题地图、范围定义清单、优先级分层、并行任务拆分、统一模板字段、假设与待确认项标记
- Can use: 以 28 天路线为主线倒推所需调研主题
- Can use: 先定义边界与问题，再进入资料检索与证据归纳
- Can use: 用表格、分层列表、问题分类框架来组织内容
- Cannot use: 脱离 28 天课程路线去做泛化的“零基础学习全景研究”
- Cannot use: 把执行重点放在具体课程内容创作，而跳过调研目标与边界定义
- Cannot use: 只罗列主题名，不说明纳入原因、排除原因或与课程路线的关系
- Cannot use: 用含糊表述替代可执行问题，如“研究一下用户需求”“看看怎么安排课程”
- Cannot use: 默认引入与当前任务无直接关系的大规模竞品分析、商业化方案、长期运营体系设计

## Dependencies and Sequence

### Milestones
1. 明确任务基准
   - 统一本次任务的唯一目标：为 zero-basics-plan 的并行综述调研建立“服务于 28 天课程路线”的目标、边界和问题清单
   - 锁定当前阶段不做的内容，如完整课程撰写、细化到每日教案、无边界竞品扩展

2. 提炼 28 天课程路线的约束条件
   - 明确 28 天意味着需要关注的时间粒度、能力节奏、任务密度与学习负担
   - 列出所有会影响调研范围的硬约束与软约束，如零基础定位、输出成果、工具门槛、连续学习要求

3. 定义“zero-basics”在本项目中的工作性含义
   - 写清楚学习者默认不会什么、可以默认会什么
   - 区分“完全无经验”“无本领域经验但有通用数字能力”等不同层次
   - 避免后续调研样本、资料筛选、课程难度设定出现偏差

4. 建立主题边界初稿
   - 从课程路线直接需要的角度列出核心主题
   - 将主题分为“必须纳入”“可选补充”“暂不纳入”
   - 为每个主题写出与 28 天课程路线的关联说明

5. 生成问题清单初稿
   - 按主题边界对应生成问题
   - 优先提出能够决定课程结构的问题，而不是泛背景问题
   - 标记高优先级问题与待后续补充问题

6. 设计并行调研拆分方式
   - 将问题清单拆成若干可独立推进的调研包
   - 统一每个调研包的记录字段、结论格式、引用要求与待确认标记
   - 确保多个调研分支最终可汇总到同一框架下

7. 进行一致性校验
   - 检查主题边界是否真正围绕 28 天课程路线
   - 检查问题是否可执行、可检索、可回答
   - 检查并行拆分后是否有遗漏、重叠或口径冲突

8. 输出最终版调研目标文档
   - 整理为可直接投入使用的结构化文档
   - 使后续执行者无需再次解释任务，即可进入并行综述调研阶段

## Implementation Notes
- 先以“28 天课程路线需要什么信息”反推调研目标，不要先发散到所有零基础学习议题
- 优先关注会影响课程成败的关键问题：对象是谁、学到哪里、每天负担多大、如何实践、如何判断完成
- 对每个主题都补一句“为什么与 28 天课程路线直接相关”，避免主题列表空转
- 问题清单建议按“人群 / 目标 / 内容 / 节奏 / 实践 / 评估 / 风险”组织
- 并行调研建议统一字段：研究问题、范围说明、关键结论、证据来源、对 28 天路线的影响、未决问题
- 可以先输出一版“主题边界 + 问题清单”骨架，再逐步补充检索维度与证据要求
- 若存在争议主题，应标记为“待确认”而不是直接并入主范围
- 课程路线相关的术语、标题、输出结构应尽量统一，避免后续综述合并时语义漂移
- 代码中不应包含 AC-、Milestone 等计划术语
