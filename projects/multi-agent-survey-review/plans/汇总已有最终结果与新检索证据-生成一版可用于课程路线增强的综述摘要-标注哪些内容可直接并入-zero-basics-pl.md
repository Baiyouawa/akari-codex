## Goal Description
整合当前项目中“已有最终结果”与“新检索证据”，重启并行综述调研流程，形成一版面向课程路线增强的结构化综述摘要。该摘要需同时满足以下用途：  
1. 可作为课程路线优化与补强的决策依据；  
2. 清晰标注哪些结论、模块、证据点可直接并入 `zero-basics-plan`；  
3. 区分“已确认结论”“新增证据支持”“仍待验证项”“不建议纳入项”；  
4. 输出格式便于后续继续扩展、审阅、追踪来源与版本迭代。  

## Acceptance Criteria

- AC-1: 已完成对已有最终结果的系统汇总，并形成统一的主题结构
  - Positive Tests: 已有最终结果被归类到明确章节或主题；不存在明显重复、冲突未标记或来源缺失的条目；每条结论都有统一命名与状态说明
  - Negative Tests: 仅做原文拼接；没有统一分类；存在多个相互冲突结论但未说明；无法分辨哪些是既有结论

- AC-2: 已完成对新检索证据的筛选、归并与可信度标注
  - Positive Tests: 新检索证据按主题映射到已有结论或新增结论；至少标明证据来源、时间、适用范围与可信度/强弱；能区分强支持、弱支持、反例或噪音
  - Negative Tests: 只罗列检索结果链接；没有筛选逻辑；无法判断证据与课程路线的关系；没有标记证据强弱

- AC-3: 已输出一版可用于课程路线增强的综述摘要
  - Positive Tests: 摘要包含课程目标、关键能力模块、推荐学习顺序、难点补强、前置依赖、风险提示等信息；可直接供课程设计者使用
  - Negative Tests: 输出仍是资料清单而非综述摘要；没有提炼出可执行的课程增强建议；无法支撑路线调整

- AC-4: 已明确标注哪些内容可直接并入 `zero-basics-plan`
  - Positive Tests: 每个可纳入项均标有“可直接并入 / 需改写后并入 / 暂不并入”等状态；并给出纳入位置建议，如章节、模块、练习、先修说明或里程碑节点
  - Negative Tests: 只说“可参考”；没有具体纳入方式；无法定位应插入 `zero-basics-plan` 的何处

- AC-5: 已显式处理冲突、空白与待验证问题
  - Positive Tests: 对证据冲突处有并列表达和判断依据；对尚无充分证据的部分列为待验证；对重要空白提出后续检索或访谈建议
  - Negative Tests: 将冲突证据强行合并；遗漏关键空白；把未验证观点当成确定结论

- AC-6: 产出具备可追踪性与可复用性
  - Positive Tests: 综述摘要中每个核心结论可追溯到已有结果或新证据；文档结构清晰，后续可继续增量更新；版本边界和更新时间明确
  - Negative Tests: 核心结论没有出处；结构混乱不便继续维护；新旧信息混杂且不可追溯

## Path Boundaries

### Upper Bound (Maximum Scope)
最完整方案包括：  
- 对已有最终结果进行全量盘点、去重、主题归类与冲突整理；  
- 对新检索证据开展并行检索、交叉验证、可信度分级与要点摘录；  
- 生成结构化综述摘要，覆盖课程目标、学习路径、能力阶梯、内容颗粒度、练习设计、常见误区、评估机制、迁移建议；  
- 对 `zero-basics-plan` 进行模块级映射，逐项标注“直接并入 / 调整后并入 / 暂不纳入”；  
- 附带证据索引表、差异说明、待补检索清单、后续迭代建议；  
- 支持后续审阅者快速定位每项结论来源与纳入理由。

### Lower Bound (Minimum Scope)
最简可行方案包括：  
- 汇总已有最终结果中的核心结论；  
- 选取与课程路线增强最相关的新检索证据进行补充；  
- 形成一版简明综述摘要；  
- 至少标出一批可直接并入 `zero-basics-plan` 的内容及其建议插入位置；  
- 对明显存在争议或证据不足的内容做最基本的风险标注。

### Allowed Choices
- Can use: 已有最终结果文档、历史调研结论、最新检索证据、主题归类表、证据强弱分级、章节映射表、并行检索与比对流程、摘要模板、版本标记
- Cannot use: 未经核实的传闻性信息、无法追溯来源的结论、仅凭个人印象的课程增强建议、跳过筛选直接拼接搜索结果、将未确认内容直接写入 `zero-basics-plan`

## Dependencies and Sequence

### Milestones
1. 明确输入范围与版本边界  
   - 收集“已有最终结果”的最新定稿范围  
   - 明确“新检索证据”的时间窗口、来源范围与筛选标准  
   - 统一综述摘要的输出模板与标注规范

2. 重启并行综述调研流程  
   - 按主题或课程模块拆分并行检索任务  
   - 建立统一记录格式，确保新证据可回填到同一框架  
   - 对重复检索、冲突结果与低质量来源进行早期清洗

3. 汇总已有最终结果  
   - 对既有结论做去重、归类、标准化命名  
   - 标记已确定、待确认、已过时、与课程无关的内容  
   - 提取可直接影响课程路线的高优先级结论

4. 整理新检索证据  
   - 对新证据进行摘要、分级与主题映射  
   - 判断其对既有结论是支持、补强、修正还是否定  
   - 记录关键出处、适用条件与限制

5. 生成课程路线增强综述摘要  
   - 按“目标—模块—顺序—补强点—风险”结构组织内容  
   - 提炼课程可执行建议，避免停留在资料层  
   - 显式区分已确认建议与需进一步验证建议

6. 标注 `zero-basics-plan` 并入建议  
   - 对每项候选内容标注处理状态  
   - 指定建议并入位置，如导学、基础概念、练习、项目、阶段复盘等  
   - 对不建议并入项给出简短理由

7. 完成最终校验与交付  
   - 检查追溯性、完整性、冲突标注与结构一致性  
   - 确认摘要可供课程路线增强直接使用  
   - 输出最终综述摘要与并入清单
