# 审查各 Worker 的证据链完整性与任务完成质量，发现偷懒或伪 blocked 立即重派

Generated: 2026-03-26

## Goal Description
建立一套可立即执行的并行综述调研审查机制，对各 Worker 当前与后续提交的任务结果进行系统复核，重点检查其证据链是否完整、任务目标是否真实完成、是否存在低质量敷衍、重复搬运、无根据结论、伪造阻塞、假装已完成或以 blocked 为名逃避推进等情况；一旦发现异常，必须在同一轮调度中快速判定、记录原因、收回任务并重派给更合适的 Worker，确保并行综述调研流程被真正重启且持续推进，而不是停留在表面状态同步。

## Acceptance Criteria

- AC-1: 已建立统一的 Worker 审查标准，并能对每个 Worker 的提交进行一致性评估
  - Positive Tests: 每个 Worker 的输出都可按统一维度检查，至少覆盖任务目标对齐、证据来源、推理过程、结论完整性、阻塞真实性、下一步行动六个方面；不同审查者依据标准能得出基本一致的判定结果
  - Negative Tests: 审查标准模糊，仅凭主观印象判断；不同 Worker 使用不同口径导致无法横向比较；无法明确区分“未完成”“低质量完成”“真实 blocked”“伪 blocked”

- AC-2: 每个 Worker 的证据链完整性都能被追踪并验证
  - Positive Tests: 任一结论都能回溯到对应来源、观察记录、处理过程和最终判断；提交中明确标注用了哪些资料、如何得出当前结论、哪些部分仍待验证；抽查时可定位关键证据
  - Negative Tests: 只有结论没有依据；引用来源但无法映射到具体结论；存在“大概看过”“参考若干资料”之类模糊表述；中间推理与最终结论断裂

- AC-3: 能有效识别偷懒、敷衍或低质量完成
  - Positive Tests: 能识别明显复述任务说明、堆砌空话、以模板冒充研究、未覆盖要求范围、只做浅层搜索、避开难点不处理、无新信息增量等行为，并给出具体判定依据
  - Negative Tests: 对明显水稿、机械拼接、格式正确但内容空洞的提交未做识别；仅因字数多就判定为高质量；无法指出偷懒的具体证据

- AC-4: 能区分真实 blocked 与伪 blocked，并对伪 blocked 立即处理
  - Positive Tests: 对声称 blocked 的 Worker，要求其提供已尝试路径、失败原因、缺失前置条件、可行替代方案和所需支持；若发现实际仍可继续推进，则标记为伪 blocked 并立即重派或要求限时补做
  - Negative Tests: Worker 仅说“资料不足”“方向不清楚”“等别人结果”就被接受为 blocked；未验证阻塞真实性；发现伪 blocked 后未采取行动

- AC-5: 存在明确的重派机制，且发现问题后能快速执行
  - Positive Tests: 对不合格任务能在同一审查周期内完成问题归类、任务回收、重写任务说明、指定新 Worker、补充上下文与截止时间；重派后能追踪新责任人是否接手
  - Negative Tests: 发现问题后只记录不处理；任务悬空无人接手；重派时未传递必要背景，导致重复返工；问题 Worker 继续占有任务但不推进

- AC-6: 并行综述调研流程已被实际重启，而非仅停留在计划层
  - Positive Tests: 已形成当前 Worker 清单、任务状态清单、审查结果清单和重派结果清单；至少一轮全量审查完成；存在新一轮并行任务分发与跟进节奏
  - Negative Tests: 只有一份计划没有执行记录；没有全量盘点当前 Worker 状态；未启动新的并行任务流转；重启仅体现在口头说明中

- AC-7: 审查结果可审计、可复盘、可用于后续质量提升
  - Positive Tests: 每个 Worker 都有审查结论、问题类型、证据摘要、处理动作、复核状态；后续可按问题类型统计高频失效点，用于优化任务描述与调度策略
  - Negative Tests: 审查过程无记录；无法说明某 Worker 为什么被判不合格或为什么被重派；事后无法复盘

## Path Boundaries

### Upper Bound (Maximum Scope)
构建完整的 Worker 质量审查与重派闭环体系：先统一任务台账与状态定义，再对所有 Worker 的历史与当前提交做全量审查；对每项任务建立“目标—证据—过程—结论—阻塞—交接”的标准化检查模板；输出逐 Worker 审查报告、问题分级、伪 blocked 判定依据、任务回收清单、优先级排序后的重派清单，以及重派后的追踪机制；同时沉淀高频偷懒模式、常见伪 blocked 话术、优质提交示例与后续抽检策略，形成可持续运行的并行综述调研治理流程。

### Lower Bound (Minimum Scope)
立即盘点所有 Worker 当前任务状态；对每个 Worker 至少检查一次最近提交，判断是否完成、是否有证据链、是否存在明显偷懒或伪 blocked；对不合格任务当场重派；形成最小可用的状态表与处理记录，保证并行综述调研流程恢复推进。

### Allowed Choices
- Can use: 任务清单、Worker 提交记录、来源链接与摘录、研究笔记、阶段性结论、阻塞说明、交接记录、统一审查表、问题分级标签、重派任务说明模板、复核日志、优先级队列
- Can use: 人工抽查与规则化检查结合；先全量快速筛查，再对高风险任务深审；按任务重要度、停滞时间、影响范围决定重派优先级
- Can use: 对真实 blocked 的任务进行拆分、补上下文、补资源或转交更合适的 Worker；对低质量但可补救的任务设置限时返工
- Cannot use: 仅凭 Worker 自述直接认定完成或 blocked
- Cannot use: 没有证据链就接受结论进入综述主线
- Cannot use: 发现问题后只做备注、不执行回收或重派
- Cannot use: 以格式完整代替内容完整；以篇幅代替质量；以忙碌为由跳过审查
- Cannot use: 在审查记录中使用含糊、不可验证的结论，如“看起来还行”“应该完成了”

## Dependencies and Sequence

### Milestones
1. 明确审查对象范围
   - 汇总全部 Worker、各自任务、当前状态、最近一次提交时间、预期产出与依赖关系
   - 标记高风险对象：长期无更新、频繁 blocked、产出空泛、关键路径任务、多人依赖任务

2. 建立统一审查框架
   - 定义审查维度：任务理解、覆盖范围、证据链、推理链、结论质量、阻塞真实性、协作交接、可复用性
   - 统一判定标签：合格、需补充、低质量返工、真实 blocked、伪 blocked、立即重派

3. 执行第一轮全量快速筛查
   - 对全部 Worker 最近提交逐一过表
   - 先识别明显完成不足、证据缺失、无效 blocked、长期停滞和高影响风险点
   - 形成初步问题清单和优先级

4. 对高风险或关键任务做深度复核
   - 抽取关键结论回溯其证据来源
   - 检查是否存在跳步推理、断章取义、只摘不判、旧结论冒充新研究等问题
   - 验证 blocked 声明是否经过充分尝试且确实受限于外部条件

5. 做出处置决策
   - 对合格项保留并进入后续综述整合
   - 对可补救项要求限时补件，明确缺失内容与截止时间
   - 对低质量或伪 blocked 项立即回收任务并重派
   - 对真实 blocked 项补充资源、拆分任务或调整依赖顺序

6. 执行重派与交接
   - 为新 Worker 提供任务目标、已有材料、前任问题、避免重复劳动的说明、时间要求和验收口径
   - 标明哪些结论不可直接继承、哪些证据可复用、哪些部分需重新验证

7. 复核重启效果
   - 确认并行综述调研已恢复多 Worker 同步推进
   - 跟踪重派任务是否真正开工、是否按新标准提交
   - 汇总第一轮审查中的高频问题，修正后续任务分发方式

## Implementation Notes
- 代码中不应包含 AC-、Milestone 等计划术语
- 审查记录应尽量结构化，建议固定字段：Worker、任务、目标、提交摘要、证据项、结论项、阻塞声明、审查结论、问题类型、处理动作、下一负责人、复核时间
- 证据链审查应关注“来源是否真实存在、内容是否被正确理解、是否足以支持结论、是否存在关键遗漏”
- 对偷懒的判断应基于可见行为证据，不应使用情绪化描述；重点记录“哪里没做、做得多浅、为什么不足以算完成”
- 对 blocked 的判断必须要求最少说明：已尝试动作、尝试结果、阻塞点、外部依赖、替代路径；缺任一关键项时，默认不能直接认定为真实 blocked
- 重派时必须补充背景上下文，避免新 Worker 从零开始；但对前任不可信部分要明确标记为“需重验”
- 若任务处于综述主干关键路径，应优先审查并优先重派，防止单点拖慢整体并行推进
- 对连续两次低质量或伪 blocked 的 Worker，应降低其后续任务复杂度或移出关键路径
- 建议采用“两层节奏”：先快速全量巡检，再针对问题任务做深审与重派，以兼顾速度和质量
- 审查输出应同时服务两个目标：一是立即恢复推进，二是为后续综述整合提供可信输入
- 所有处理动作都应可追踪，确保任务从发现问题到重派接手之间没有无人负责的空档
