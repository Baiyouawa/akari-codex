- 适用人群: 零基础
- 难度标签: 入门
- 推荐原因:
  - 对 `Host`、`HostName`、`Port`、`User` 有逐项解释

### 1.4 VSCode 模块教学建议

#### 适合零基础的讲法

- 第一步不要先讲“SSH 协议定义”，先讲：**你在自己电脑上打开 VSCode，但实际上编辑的是服务器里的文件**。
- 再解释：
  - **Remote-SSH** = 让 VSCode 像“远程控制器”一样连到另一台电脑
  - **SSH** = 安全的远程通道
  - **密钥登录** = 用钥匙，不用每次输门禁密码
  - **SSH config** = 远程服务器通讯录

#### 教学上的最小闭环

1. 安装 Remote-SSH 插件
2. 先用密码连通一台服务器
3. 打开远程文件夹
4. 再做密钥免密
5. 最后讲常见故障：端口、用户名、密钥路径、主机指纹

- 适用人群: 学完基本导航后、准备理解系统结构的新手
- 难度标签: 入门
- 推荐原因:

- 主题覆盖:
  - 文件类型位
  - `rwx` 权限模型
  - `chown`、`chmod`
  - 所有者、所属组、其他用户
- 适用人群: 准备学习权限的新手
- 难度标签: 入门到初级
- 推荐原因:
  - 对 `ls -l` 输出拆得比较细，适合做课程图解
- 风险与备注:
  - 权限内容容易让新手一下接触太多术语，应拆成“读/写/执行”和“谁有权限”两层讲
- 零基础解释方式:
  - **生活类比**：权限就是“谁可以看、改、运行这个东西”。
  - **最小示例**：先只看一行 `-rw-r--r--`，不要同时讲特殊权限位。
  - **常见错误**：把目录的执行权限和文件的执行权限当成一回事。

## 四、对零基础最友好的解释模板

后续写 28 天教程时，建议每个知识点都尽量用同一种模板讲：

### 模板 A：概念解释

- **这是什么**：一句人话定义
- **它解决什么问题**：为什么你需要它
- **把它想成什么**：生活类比
- **最小可运行示例**：一条命令或一个最短配置
- **常见错误**：新手最容易踩的坑

### 示例

#### SSH
- 这是什么：一种安全地登录远程电脑的方法。
- 它解决什么问题：你不用坐在服务器前，也能操作它。
- 把它想成什么：像通过加密门禁进入远处机房。
- 最小示例：`ssh user@server_ip`
- 常见错误：用户名写错、端口不对、服务器没开 SSH。

#### `cd`
- 这是什么：切换当前所在目录。
- 它解决什么问题：你要先“走到地方”，才能操作里面的文件。
- 把它想成什么：在文件夹房间之间走动。
- 最小示例：`cd /home/ubuntu`
- 常见错误：把文件名当目录名、把相对路径和绝对路径混淆。

#### `chmod`
- 这是什么：改文件或目录权限。
- 它解决什么问题：控制谁能看、谁能改、谁能执行。
- 把它想成什么：给门设置“谁能进、谁能改、谁能操作设备”。
- 最小示例：`chmod u+x script.sh`
- 常见错误：动不动就 `chmod 777`。

## 六、关键风险点与交叉校验

### 6.1 版本与平台风险

- **VS Code / Remote-SSH 版本差异**
  - 依据: VS-01、VS-03
  - 风险: `.vscode-server` 目录、离线安装方式、兼容性提示可能变化
  - 处理: 课程中对“离线安装/清理 known_hosts/降级版本”统一放到排错附录，不放正文主路径

- **Windows 与 Linux/macOS 路径差异**
  - 依据: VS-02、VS-05、LI-02
  - 风险: 私钥路径写法、命令可用性不同
  - 处理: 教程示例默认给 Linux/macOS，再单列 Windows 注释

### 6.2 危险命令风险

- `rm -rf`
  - 依据: LC-02 中出现强制删除示例
  - 风险: 新手误删系统文件
  - 处理: 只在 `~/sandbox` 等测试目录演示

- `chmod -R` / `chown -R`
  - 依据: LI-05、LC-04、GNU chmod 手册
  - 风险: 递归误改权限；GNU 手册明确提到递归处理符号链接存在安全风险
  - 校验: `web_fetch(https://www.gnu.org/software/coreutils/manual/html_node/chmod-invocation.html)`

- `kill -9`
  - 依据: LC-01
  - 风险: 新手把它当通用结束方式
  - 处理: 课程中强调先学普通 `kill`，再讲 `-9` 是更强制的最后手段

### 6.3 配置校验点

- `~/.ssh/config` 的字段含义与优先顺序
  - 校验来源: `web_fetch(https://man.openbsd.org/ssh_config)`
  - 关键点: 用户配置文件 `~/.ssh/config` 与系统级配置文件 `/etc/ssh/ssh_config` 都可能生效；更具体的 `Host` 配置应放前面

- VS Code Remote-SSH 的官方术语与安装路径
  - 校验来源: `web_fetch(https://code.visualstudio.com/docs/remote/ssh)`
  - 用途: 校验经验帖中的插件名、工作流和术语
