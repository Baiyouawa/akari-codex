## 四、学习顺序说明

### 4.1 总体顺序

建议采用以下顺序：

1. **先建最小操作能力**：终端、路径、文件、权限、SSH、VSCode 远程连接。
2. **再建协作能力**：Git 本地版本控制、GitHub 仓库协作、基础工作流。
3. **再建生产力能力**：Cursor、AI 辅助编程、提示词、调试与 Agent 协作。
4. **最后做知识迁移与综合实践**：将前面的环境、协作、AI 辅助能力迁移到基础深度学习入门与小型项目中。

### 4.2 这样安排的原因

- `analysis/2026-03-26-vscode-linux-beginner-resource-survey.md` 已证明 Linux / SSH / VSCode 远程开发适合作为零基础起点，因为它们能建立“远程电脑、终端、路径、文件、权限”的最小认知闭环。
- Git / GitHub 放在第二阶段，原因是学习者先需要具备基本文件和目录操作能力，才能真正理解仓库、提交、分支、合并这些抽象概念。
- Cursor / AI 辅助编程放在第三阶段，原因是学习者先要会读文件、跑命令、使用 Git，AI 工具才不会变成黑箱。
- 深度学习放在第四阶段，原因不是它不重要，而是前 3 周先建立“环境 + 协作 + 工具”后，学习者已经具备阅读教程、运行代码、记录实验、排查错误的基础能力。

### 4.3 每周内部顺序

每周都遵循同一节奏：

- Day 1-2：概念和环境
- Day 3-5：核心操作与单点练习
- Day 6：串联式任务
- Day 7：系统课程（回顾 + 知识地图 + 综合练习 + 阶段验收）

### 10.1 标题层级规范

- `##`：总模块 / 周模块 / Day 模块
- `###`：固定字段
- `####`：正文内部小节

### 10.2 字段命名规范

所有 Day 文档统一使用以下字段名：

- 今日定位
- 学习目标
- 前置知识
- 学习顺序
- 核心概念
- 教程正文
- 今日任务安排
- 实践内容
- 常见问题
- 当日成果检查
- 当日总结
- 资料来源
- 延伸资料

系统课程额外增加：

- 本周回顾目标
- 本周知识地图
- 综合练习
- 阶段成果检查
- 下周衔接说明

### 10.3 列表与表格规范

- 所有验收勾选统一使用 `- [ ]`
- 所有来源统一使用 Markdown 表格，不混用自由文本
- 所有命令示例统一使用 fenced code block
- 所有风险项统一放在“风险提示”小节，不散落正文

## 十一、适用于 Multi-Agent 的拆分方式

### 11.1 可并行模块

1. **结构设计 Agent**
   - 负责主文档目录、模板、字段规范
2. **资料调研 Agent**
   - 分主题调研 Linux / Git / Cursor / 深度学习资料
3. **日课编写 Agent**
   - 按天填充模板内容
4. **来源核验 Agent**
   - 统一检查链接、出处、可信度说明
5. **系统课程 Agent**
   - 专门负责编写 Day 07 / 14 / 21 / 28
6. **统稿 Agent**
   - 合并各模块，统一格式、术语和难度

### 11.2 合并接口

所有 Agent 的输出必须满足以下接口约束：

- 文件命名统一：`day-01.md`、`day-02.md`...
- 字段顺序统一，不得自由重排
- 每个 Day 文件结尾必须有“资料来源”表
- 每个 Day 文件开头必须有“今日定位”
- 系统课程必须额外带“知识地图”和“综合练习”

### 11.3 质检节点

- 日课 Agent 完成后：检查字段是否齐全
- 来源核验后：检查链接和用途字段是否完整
- 统稿前：检查周内顺序是否合理
- 主文档合并后：检查 28 天编号与周映射是否连续

## 十二、最小可执行交付流程

1. 先建立主文档空骨架。
2. 再建立 4 个周概览文件。
3. 同时创建 28 个 Day 空模板文件。
4. 资料调研 Agent 按主题回填来源表。
5. 日课编写 Agent 按模板填正文。
6. 系统课程 Agent 单独填 4 个系统课程。
7. 来源核验 Agent 补齐来源用途和可信度。
8. 统稿 Agent 合并为 `zero-basics-complete-tutorial.md`。
