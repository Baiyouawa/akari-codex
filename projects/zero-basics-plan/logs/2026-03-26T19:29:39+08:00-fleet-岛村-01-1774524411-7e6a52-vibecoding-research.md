## Findings

- Google Cloud 将 vibe coding 描述为从逐行写代码转向通过对话式方式引导 AI 生成、完善、调试应用，并将其区分为 “Pure vibe coding” 与 “Responsible AI-assisted development”；后者强调用户需要 review、test、understand 生成的代码并承担最终责任。来源：
  - https://cloud.google.com/discover/what-is-vibe-coding
- IBM 将 vibe coding 描述为用户用 plain speech 表达意图，AI 把 thinking 转换为 executable code，并在开发环境中作为实时 coding assistant 提供建议。来源：
  - https://www.ibm.com/think/topics/vibe-coding
- 对零基础课程，最稳妥的教学版本不是“放手让 AI 写”，而是“AI 先搭最小版本，人类负责运行、验证、逐步迭代”。这与 Google Cloud 的 “Describe the goal → AI generates code → Execute and observe → Provide feedback and refine → Repeat” 一致。来源：
  - https://cloud.google.com/discover/what-is-vibe-coding
- 提示词基础的可教版本是“目标 + 上下文 + 约束 + 输出格式 + 验证方式”，而不是复杂咒语；OpenAI 强调 explicit instructions，Anthropic 则明确将 clarity、examples、XML structuring、agentic systems 列为最佳实践。来源：
  - https://developers.openai.com/api/docs/guides/prompt-engineering
  - https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices
- Cursor 官方资料可直接映射为教学模块：Rules/AGENTS.md 负责持续约束，Terminal 负责执行与 sandboxing，Developing Features 强调 plan / test-driven / trust，Debug Mode 强调 hypothesis generation / log instrumentation / runtime analysis。来源：
  - https://cursor.com/docs/rules
  - https://cursor.com/docs/agent/tools/terminal
  - https://cursor.com/learn/creating-features
  - https://cursor.com/docs/agent/debug-mode
