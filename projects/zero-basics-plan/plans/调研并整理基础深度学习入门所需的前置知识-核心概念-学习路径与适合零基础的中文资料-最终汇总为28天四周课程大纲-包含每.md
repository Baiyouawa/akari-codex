## Implementation Notes
- 先搭建统一的信息采集模板，再开始资料收集，避免后期资料格式不一致
- 前置知识建议按“必须补齐”和“边学边补”拆分，减少零基础用户的心理负担
- 核心概念讲解应以直觉解释优先，数学推导作为辅助，而不是主体
- 学习路径设计应坚持“先理解训练流程，再接触模型变体”的原则
- 中文资料优先考虑稳定可访问、更新较近、讲解口语化且示例明确的内容
- 每周系统课建议承担“串联本周知识”和“纠正常见误区”的作用，而不是重复每日任务
- 每天任务应控制在零基础可完成的范围内，建议包含输入型任务与输出型任务各至少一种
- 课程大纲中应预留复盘日、练习日或轻量任务日，防止连续高强度输入导致掉队
- 28 天课程最好形成一条主线：认识 AI 与深度学习 → 补齐基础 → 理解神经网络训练 → 完成简单实践 → 建立后续进阶方向
- 整理资料时建议标注“适合第几天/第几周使用”，方便与课程大纲直接映射
- 若某主题优质中文资料不足，可采用“中文讲解 + 官方文档摘要”的组合方式，但主体仍应保持中文友好
- 输出内容应能直接支持后续撰写完整 Markdown 教程，因此章节命名应简洁、一致、可扩展
- 代码中不应包含 AC-、Milestone 等计划术语
