# 28 天《零基础计划》课程框架设计

- Date: 2026-03-26
- Session: `岛村-01-1774523931-34903a`
- Scope: 4 周、28 天、零基础、Markdown 教程总框架
- Status: draft-ready

## 设计目标

1. 用 4 周把学习路径拆成可连续执行的 28 天日程。
2. 每天都包含：学习目标、实操任务、配套资料、阶段产出。
3. 每周末固定安排一次系统复盘与整合训练，避免只学概念不做整合。
4. 课程顺序遵循“环境与命令行 → 版本控制与协作 → Python 与 AI 编程 → 深度学习入门与综合项目”的依赖关系。

## 设计依据

- Linux 命令行入门优先，因为后续 Git、Python、包管理、环境配置都依赖终端基本操作。
- Git 与 GitHub 放在第 2 周，便于尽早形成版本管理与协作习惯。
- Cursor / VS Code / VibeCoding 安排在第 3 周，与 Python 基础同步推进，避免工具学习脱离实际编码任务。
- NumPy 与 PyTorch 基础放在第 4 周，因为其先修依赖是 Python、环境配置与基本调试能力。
- 每周第 7 天固定为复盘/整合训练，因此 4 周共 `4 × 7 = 28` 天，其中复盘日共 `4` 天、主题学习日共 `24` 天。

## 周结构

### Week 1 — Linux 与终端基础

目标：建立文件系统、命令行、权限、Shell 与软件安装的最低可用能力。

### Week 2 — Git / GitHub 与协作基础

目标：理解版本控制、远程仓库、分支协作、PR 与 issue 流程。

### Week 3 — Python + Cursor + VibeCoding 基础实践

目标：建立编程基本功，并学会用 AI 编程工具完成小任务而不是盲目生成代码。

### Week 4 — NumPy / PyTorch / 深度学习入门与综合项目

目标：理解张量、训练、推理、自动求导与最小深度学习 demo，并完成一次综合交付。

## 每日产出模板

每一天默认包含以下四项：

1. 学习目标
2. 实操任务
3. 配套资料
4. 阶段产出

最终教程可直接按该模板展开为 28 个 Markdown 小节。
