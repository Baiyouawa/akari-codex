## Goal Description
产出一份可直接交付、结构统一、适合连续 28 天学习的 Markdown 教程文档。该教程需基于多平台资料并行调研后完成整合，覆盖以下核心内容：

- 按天拆分 28 天学习计划，每天都包含：
  - 学习目标
  - 任务安排
  - 操作步骤
  - 推荐资料
  - 来源链接
- 按周设计周末系统课程内容
- 每周配置复盘作业与总结任务
- 全文结构统一、表达风格一致、难度递进合理
- 推荐资料具备可追溯来源，尽量覆盖不同平台与内容形态
- 最终以单一 Markdown 文档交付，并具备良好的目录层级与可读性

该任务本质上包含“资料调研—筛选归类—课程编排—内容撰写—格式统一—质量校验—最终整合”七个阶段，目标是形成一份既能直接学习，也能继续扩展维护的教程大纲与正文初稿。

## Acceptance Criteria

- AC-1: 最终交付物为一份完整的 Markdown 教程文档，包含 28 天逐日内容与周末模块
  - Positive Tests: 文档中清晰标注 Day 1 至 Day 28；每一天均包含学习目标、任务安排、操作步骤、推荐资料、来源链接；每周末包含系统课程与复盘作业
  - Negative Tests: 仅有大纲没有逐日内容；缺少任意一天；周末部分未单独设计；内容散落在多个文件中未整合为单一 Markdown

- AC-2: 教程内容基于多平台资料调研结果整合，而非单一来源拼接
  - Positive Tests: 推荐资料来自多个平台，如官方文档、视频平台、技术社区、课程平台、博客或书籍站点；每项资料有明确来源链接
  - Negative Tests: 全部内容仅引用单一平台；无法追溯资料出处；来源链接缺失或失效比例过高

- AC-3: 28 天学习路径具备循序渐进的难度设计和明确的周目标
  - Positive Tests: 第 1 周偏基础认知与环境准备，第 2-3 周进入核心技能与实践，第 4 周进入整合应用、复盘与输出；每天内容前后依赖合理
  - Negative Tests: 高难度主题在前期集中出现；后续内容与前置知识脱节；学习顺序混乱

- AC-4: 每日任务具备可执行性，操作步骤足够具体，学习者可按文档直接执行
  - Positive Tests: 每日都含具体执行步骤，如“阅读—记录—练习—提交—总结”；任务颗粒度适中；单日工作量可控
  - Negative Tests: 只写抽象目标，不写执行步骤；步骤过于粗略，无法落地；单天任务明显超负荷

- AC-5: 推荐资料经过筛选与分类，避免重复、失焦或质量过低
  - Positive Tests: 每天推荐资料数量适中；资料与当天学习目标高度相关；同类资源有主次区分；必要时标注“必读/选读”
  - Negative Tests: 大量重复链接；资料与当天主题无关；堆砌资源但未说明用途；低质量内容占比过高

- AC-6: 周末系统课程内容不是简单重复日常任务，而是对一周内容进行体系化整理
  - Positive Tests: 周末模块包含知识结构梳理、典型案例、常见误区、阶段总结、集中练习或小项目
  - Negative Tests: 周末仅复制前几天任务；没有体系化讲解；没有复盘作业或输出要求

- AC-7: 文档格式统一，Markdown 层级清晰，便于阅读、协作与后续维护
  - Positive Tests: 标题层级一致；列表样式统一；链接格式规范；资源引用位置稳定；目录结构明确
  - Negative Tests: 标题级别混乱；同类内容使用不同模板；链接书写风格不统一；段落结构杂乱

- AC-8: 教程内容应体现调研、整合与编辑，而非原文照搬
  - Positive Tests: 对资料进行摘要、重组、归纳与课程化编排；不同来源内容被统一成一致的教学表达
  - Negative Tests: 大段复制原始资料；仅做链接汇总；缺少整理后的教程化内容

- AC-9: 最终文档需经过基本质量检查
  - Positive Tests: 链接可访问率高；重复内容已去除；术语一致；日期与天数无错位；章节无明显缺漏
  - Negative Tests: 存在大量死链；Day 编号错误；术语混乱；内容前后矛盾；遗漏复盘作业或来源链接

## Dependencies and Sequence

### Milestones
1. 明确教程主题边界与目标学习人群
   - 确认这份 28 天教程服务的对象、起点水平、最终能力目标
   - 明确教程范围，避免后续调研失焦
   - 形成“内容边界说明”和“学习成果定义”

2. 设计统一的 Markdown 教程模板
   - 先定义整篇文档目录结构
   - 统一每天的固定字段：
     - 学习目标
     - 任务安排
     - 操作步骤
     - 推荐资料
     - 来源链接
   - 统一每周末的固定字段：
     - 系统课程内容
     - 知识框架
     - 复盘作业
     - 阶段检查
   - 输出模板样例，作为后续批量写作规范

3. 建立资料调研清单与来源分类规则
   - 将资料来源按平台类型分类
   - 设定质量评估标准，如权威性、时效性、可读性、难度、是否适合新手
   - 设定去重规则与引用规则
   - 明确哪些资料优先作为主线材料，哪些仅作为补充

4. 并行调研多平台资料
   - 按 28 天主题框架初步拆分调研任务
   - 对每个主题分别搜集可用资料
   - 记录资料名称、类型、适用阶段、链接、摘要、使用建议
   - 初步筛除低质量和重复资料

5. 形成 4 周学习结构与日程骨架
   - 先从周视角设计阶段目标
   - 再细化到每日主题与顺序
   - 确保知识依赖关系合理
   - 为周末预留体系化讲解与复盘位置

6. 完成 28 天逐日内容初稿
   - 逐天填写学习目标、任务安排和操作步骤
   - 为每一天分配最合适的推荐资料和来源链接
   - 标注当日重点、预期产出与注意事项
   - 保证单日内容量均衡

7. 完成每周末系统课程与复盘作业设计
   - 将一周内容提炼为系统课程
   - 设计综合性练习、问答题、总结任务或小项目
   - 增强知识串联，而非简单重复
   - 确保复盘作业能检查学习效果

8. 全文整合与风格统一
   - 合并调研结果与逐日内容
   - 统一术语、措辞、标题层级、资料标注格式
   - 清理重复表述与断裂内容
   - 补齐缺失链接和说明项

9. 质量检查与交付定稿
   - 检查 28 天编号是否完整
   - 检查每天字段是否齐全
   - 检查周末模块是否完整
   - 抽查链接有效性
   - 完成最终 Markdown 文档定稿

## Implementation Notes
- 先搭结构，后填内容，避免边调研边改框架造成返工
- 以“周目标—日主题—周末系统课”三级结构组织内容，便于控制节奏
- 每天的学习时长建议隐含控制在可执行范围内，避免出现极端不均衡的任务量
- 推荐资料应注明用途，如“入门概念”“操作演示”“进阶补充”“案例参考”
- 对来源链接建议统一格式，避免正文中出现混乱的裸链风格
- 同一主题若存在多个高质量来源，应明确主次，不必全部堆入正文
- 周末系统课程应承担“归纳、串联、强化、检测”功能
- 复盘作业优先设计为可输出成果的形式，如总结、清单、练习、案例复现、小项目
- 多平台资料整合时，应优先保留权威、时效性高、表达清晰的来源
- 对于可能变化较快的外部资料，应尽量选择稳定入口页而非临时页面
- 写作时保持固定模板，减少后期统一格式的成本
- 最终文档建议增加目录，便于按周、按天快速跳转
- 若资料量过大，可先建立候选资源池，再进行二轮筛选
- 全文应面向学习者阅读体验，而不是面向资料收集过程展示
- 代码中不应包含 AC-、Milestone 等计划术语
