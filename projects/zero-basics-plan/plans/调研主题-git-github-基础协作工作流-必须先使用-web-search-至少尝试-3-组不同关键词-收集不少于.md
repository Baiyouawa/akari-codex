## Implementation Notes
- 先搜索，再筛选，再归纳，再写 Markdown 草稿，不可颠倒顺序
- 搜索关键词应尽量分组明确，避免 3 组关键词只是同义重复
- 中文高质量来源优先级建议为：官方中文文档/官方帮助中心 > 高质量教程站点 > 知名中文技术社区优质文章
- 若某一主题资料不足，应补充定向搜索，而不是用弱相关内容填充
- “常见报错与解决”建议重点关注：
  - SSH key/Permission denied
  - push 被拒绝
  - pull/merge 冲突
  - remote origin 已存在
  - 仓库初始化或分支不一致
- “适合插入第几天课程”建议采用递进式安排，例如：
  - 第 1 天：Git 基础概念、add/commit
  - 第 2 天：仓库创建、remote、push/pull、SSH
  - 第 3 天：branch/merge、协作流程
  - 第 4 天：常见报错与排查
- Markdown 调研笔记草稿建议包含以下结构：
  - 调研说明
  - 搜索关键词记录
  - 资料清单
  - 按课程天数整理的知识点笔记
  - 参考链接汇总
- 每条摘要应控制为“能帮助课程设计者快速判断用途”的信息密度，而不是简单复制标题
- 若同一站点多篇文章内容高度重合，应择优保留信息更完整、更新时间更合理的一篇
- 最终草稿中应能看出“资料来源”与“课程内容”的对应关系
- 代码中不应包含 AC-、Milestone 等计划术语
