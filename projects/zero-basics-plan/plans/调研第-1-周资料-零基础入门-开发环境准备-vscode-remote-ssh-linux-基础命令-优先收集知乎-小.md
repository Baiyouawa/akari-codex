## Goal Description
围绕“第 1 周学习内容”完成一份可直接用于后续 28 天完整教程编写的资料调研与整理计划。调研范围覆盖以下四个主题：零基础入门、开发环境准备、VSCode Remote SSH、Linux 基础命令。任务重点是并行收集多平台、较新且质量较高的中文资料，优先覆盖知乎、小红书、技术博客、优质视频/帖子，并对每条资料记录来源、发布时间、适用对象、核心内容、可信度与是否过时。最终产出应支持团队后续共同整理为教程大纲与 Markdown 文档，并确保资料具备可追溯性、可筛选性和可复用性。

## Acceptance Criteria

- AC-1: 完成第 1 周四大主题的资料覆盖与分类整理
  - Positive Tests: 四个主题均至少有独立资料清单；每个主题包含基础概念、操作步骤、常见问题或避坑点；资料可被明确归类到对应主题
  - Negative Tests: 仅覆盖部分主题；资料混杂未分类；无法判断某条资料属于哪个主题

- AC-2: 优先收集非陈旧资料，并记录完整来源信息
  - Positive Tests: 每条资料至少记录标题、平台、作者/发布者、链接、发布时间或更新信息、摘要；优先选择近 2 年内内容，较早资料需注明“经典但可能过时”的风险说明
  - Negative Tests: 资料无链接或无来源；大量使用明显过时内容；无法判断发布时间；转载内容未追溯原始出处

- AC-3: 多平台资料结构均衡，符合“知乎、小红书、博客、优质视频/帖子优先”的要求
  - Positive Tests: 最终清单中包含上述优先平台中的多个来源类型；每个主题至少覆盖 2 种不同平台形态；视频与图文资料均有体现
  - Negative Tests: 只依赖单一平台；优先平台基本缺失；全部为搜索引擎结果页而非原始内容

- AC-4: 对资料质量进行初步评估，能支撑教程编写
  - Positive Tests: 每条资料附带适用人群、可信度判断、内容完整度、实操性、是否适合纳入教程；能识别营销软文、内容农场和明显抄袭内容
  - Negative Tests: 仅做链接堆砌；没有质量判断；低质量内容与优质内容未区分；无法为后续教程筛选提供依据

- AC-5: 形成可直接转入 Markdown 文档编写的结构化输出
  - Positive Tests: 产出包含主题目录、推荐资料表、筛选结论、建议采用顺序、待补充空缺点；字段统一，便于复制到 Markdown 文档
  - Negative Tests: 资料格式混乱；字段不一致；后续需要重新清洗才能写教程

- AC-6: 明确并行调研分工与汇总顺序，保证团队协作可执行
  - Positive Tests: 为各主题或平台设置并行调研路径；定义汇总、去重、复核与定稿流程；有统一命名和记录规范
  - Negative Tests: 没有分工建议；多人产出难以合并；重复搜集严重；无法判断谁负责复核

## Path Boundaries

### Upper Bound (Maximum Scope)
完成一份高质量的第 1 周资料库与整理方案，覆盖四大主题下的核心知识点、安装与配置步骤、典型错误案例、平台对比、版本差异、适用人群分层建议。每个主题均收集不少于 8–12 条候选资料，来源横跨知乎、小红书、个人博客、掘金/CSDN/公众号、B 站/视频平台、GitHub 文档、官方文档等；对每条资料记录完整元数据并做可信度评分、时效性判断、内容摘要、引用建议、是否适合新手。最终形成一版可直接支撑“28 天完整教程大纲”和“第 1 周 Markdown 文档”编写的调研底稿，并附去重规则、资料纳入/剔除理由、内容缺口清单与后续补充建议。

### Lower Bound (Minimum Scope)
完成四大主题的基础资料调研，每个主题至少筛选 3–5 条可用资料；每条资料至少记录标题、平台、链接、发布时间、适用对象、简要摘要和是否推荐；确保优先平台有覆盖，并形成一版基础 Markdown 结构草稿，能支持后续继续扩展。

### Allowed Choices
- Can use: 搜索引擎检索、平台站内搜索、官方文档、知乎、小红书、技术博客、优质视频平台、开发者社区帖子、GitHub README/Issues 中的高质量讨论、Wayback/更新时间辅助判断时效性的工具、共享表格或 Markdown 表格进行汇总
- Cannot use: 无法追溯来源的二次转载、纯 AI 生成但无原始出处的内容聚合页、明显过时且未标注风险的教程、盗版课程资源、需要付费但无法核验内容质量的封闭资源、仅有标题党无实质内容的帖子、无发布时间且无证据可判断新旧的资料作为核心依据

## Dependencies and Sequence

### Milestones
1. 明确第 1 周四大主题的调研框架与字段模板
   - 确定四个主题的边界：零基础入门、开发环境准备、VSCode Remote SSH、Linux 基础命令
   - 统一记录字段：标题、平台、作者、链接、发布时间、最后更新时间、主题标签、适用人群、摘要、可信度、实操性、时效性、推荐等级、纳入建议、备注
   - 明确“优先平台”和“次级补充平台”的检索顺序

2. 建立检索策略与筛选规则
   - 为每个主题准备关键词组合，例如“VSCode Remote SSH 新手 配置 2025”“Linux 基础命令 入门 教程”“开发环境准备 Python/Node/Git 新手”
   - 明确时效性优先级：优先近 2 年，其次近 3–4 年内的经典内容
   - 定义排除条件：广告密集、内容浅薄、无操作截图、版本信息缺失、明显抄袭、评论区大量指出错误

3. 并行收集多平台候选资料
   - 按主题并行：每人负责 1 个主题，或按平台并行：每人负责 1 类平台
   - 先采集优先平台资料，再补充官方文档与高质量社区帖子
   - 对每条候选资料进行初步摘要和可用性判断，避免只存链接不做备注

4. 去重、复核与补齐缺口
   - 合并重复链接、重复转载和内容高度相似的资料
   - 检查四大主题是否均有“概念解释 + 操作步骤 + 常见问题/避坑”三类内容
   - 若某主题存在明显缺口，补充官方文档、GitHub 讨论或高质量视频说明

5. 形成第 1 周资料推荐清单
   - 每个主题输出“核心推荐资料”“备选资料”“不推荐资料及原因”
   - 为后续教程撰写给出建议阅读顺序与知识递进关系
   - 将资料按新手友好度排序，便于后续组织成课程式结构

6. 产出可用于 Markdown 文档和 28 天教程大纲的调研结果
   - 输出统一格式的资料表与主题概述
   - 给出第 1 周教程建议章节结构
   - 标注待确认事项、易过时点和后续需要持续更新的内容
