## Acceptance Criteria

- AC-1: 输出一份覆盖完整主题范围的调研清单
  - Positive Tests: 至少覆盖神经网络、训练与推理、损失函数、过拟合、数据集、PyTorch、基础实践 7 个主题；每个主题下至少有 2 份候选资料
  - Negative Tests: 缺少任一核心主题；某主题只有单一资料来源；资料仅覆盖理论但没有实践内容

- AC-2: 所有资料均适合“完全新手”定位
  - Positive Tests: 资料明确面向初学者；术语解释充分；前置知识要求低；学习路径从概念到实践逐步展开
  - Negative Tests: 资料默认读者已掌握线性代数、微积分、机器学习基础；大量使用高级公式但无解释；直接进入复杂项目实现

- AC-3: 每条资料记录都包含完整来源信息
  - Positive Tests: 每条资料至少包含标题、平台/作者、链接、发布时间或版本信息（若可得）、语言、主题标签、推荐理由
  - Negative Tests: 只有资料名称没有链接；无法判断出处；引用来源模糊为“网上看到”或“某教程”

- AC-4: 对资料进行质量筛选与分级
  - Positive Tests: 每条资料标注难度、形式（视频/文章/课程/文档/代码）、是否建议必学、预估学习时长、适合放入第几天或第几周
  - Negative Tests: 仅做资料堆砌，不区分优先级；无法支持 28 天教程编排；没有学习顺序建议

- AC-5: 形成可直接用于后续编写 Markdown 教程的结构化结果
  - Positive Tests: 输出包含按天或按周的知识递进建议；每个阶段有学习目标、资料组合、练习建议与产出要求；结构适合直接扩展成教程正文
  - Negative Tests: 输出只是零散书单；没有章节结构；无法映射到 28 天学习计划

- AC-6: 覆盖多个内容平台并适合 Multi-Agent 并行执行
  - Positive Tests: 至少覆盖 4 类平台，如官方文档、课程平台、技术社区、视频平台、开源代码仓库、博客专栏；任务可拆分给不同 Agent 并最终汇总
  - Negative Tests: 所有资料都来自单一平台；计划无法并行；不同 Agent 的产出标准不一致导致无法合并

- AC-7: 包含基础实践与可运行的 PyTorch 入门内容
  - Positive Tests: 至少包含环境准备、Tensor 基础、Dataset/DataLoader、简单前向网络、训练循环、验证/推理、小型练习项目等实践内容
  - Negative Tests: 只有 PyTorch API 列表；没有从零实现一个最小训练示例；缺乏新手可执行练习

- AC-8: 明确排除超出零基础范围的内容
  - Positive Tests: 对高级主题如 Transformer 细节、分布式训练、复杂数学推导、工业级部署等标注为非本阶段重点或可选扩展

   - 推荐理由
