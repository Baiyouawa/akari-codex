1. 对资料进行分类整理，包括但不限于：
   - VSCode Remote SSH 基础概念
   - SSH 连接前置准备
   - Windows / macOS / Linux 本地环境配置
   - 远程 Linux 服务器基础认知
   - 常见连接方式与配置
   - 报错排查与高频问题解决
   - 初学者实践任务与演练案例

2. 为每条资料标注结构化元信息，包括：
   - 标题
   - 来源平台
   - 原始链接
   - 作者 / 发布者
   - 发布时间（若可获取）
   - 内容类型（图文 / 视频 / 教程 / 经验帖 / 问答）
   - 适用人群
   - 难度等级

3. 基于整理后的资料，设计适合零基础学习者的 28 天课程结构，要求：

   - 每天有明确主题、学习目标、推荐资料、实践任务
   - 能支持后续由多代理协作继续扩写为完整教程

4. 最终沉淀为一份完整 Markdown 教程骨架，满足：
   - 来源可追溯
   - 结构清晰
   - 便于多角色并行补充

## Acceptance Criteria

- AC-1: 完成面向零基础学习者的资料调研清单，覆盖目标主题的核心学习路径
  - Positive Tests: 至少形成一份包含“环境准备、SSH 基础、VSCode Remote SSH 安装与配置、首次连接、常见报错、文件上传下载、基础命令行操作、安全注意事项”模块的资料总表
  - Negative Tests: 仅收集零散链接，没有覆盖完整学习链路；或缺少连接前置条件与常见报错内容

- AC-2: 资料来源符合优先级要求，并对来源进行明确标注
  - Positive Tests: 清单中优先包含知乎、小红书、个人博客、实战教程；每条资料均标注来源平台与原始链接
  - Negative Tests: 大量使用泛化搜索结果页、无作者信息的搬运站、无法追溯原始出处的二次转载内容；或未标注来源平台

- AC-3: 每条资料都完成适用人群与难度判断，适合后续教学筛选
  - Positive Tests: 每条资料至少标注“适合完全零基础 / 适合有基础命令行经验 / 适合遇到报错排障者”等标签，并附简要推荐理由
  - Negative Tests: 仅给出链接，不区分适用对象；或所有资料都被笼统标注为“适合初学者”而无细分依据

- AC-4: 对资料进行质量筛选与风险标注
  - Positive Tests: 能识别内容是否过时、是否依赖旧版 VSCode 插件、是否使用已变化的服务器配置方式、是否缺少安全提醒，并做显式备注
  - Negative Tests: 将明显过时或步骤失效的资料直接纳入推荐列表，且不做说明

- AC-5: 输出结构化 Markdown 文档，支持继续扩展与多代理协作
  - Positive Tests: 最终文档包含统一字段模板、分类目录、资料表格或条目清单、课程结构草案、待补充事项，便于不同 Agent 分工补写
  - Negative Tests: 输出为非结构化随笔式总结，无法拆分模块，无法继续协作加工

- AC-6: 形成一套可执行的 28 天课程结构草案
  - Positive Tests: 课程按周或按天分层设计，包含每日目标、所需资料、练习任务与阶段性成果；整体难度从零基础逐步进阶
  - Negative Tests: 仅有课程名称列表，没有每日学习目标与配套资料；或课程难度跳跃过大，不适合零基础

- AC-7: 明确区分“资料调研成果”与“教程成稿骨架”
  - Positive Tests: 文档中至少分为“资料库”“筛选推荐”“28 天课程结构”“完整教程目录草案”四个层次
  - Negative Tests: 将原始资料、课程设计、最终教程内容混杂在一起，导致后续无法复用

- AC-8: 输出结果具有最小可用性，可直接进入下一阶段内容生产
  - Positive Tests: 即使未扩写完整正文，也至少能让后续 Agent 按模块继续补充，如“连接前准备”“Windows 配置”“SSH 密钥配置”“报错排查案例”等
  - Negative Tests: 只有高层计划，没有任何可执行目录、模板、字段规范或资料样例
