## Implementation Notes
- 建议优先建立“主题 × 平台 × 难度 × 可信度”的四维标签体系，避免后期资料堆积后难以课程化整理。
- 资料模板建议统一为 Markdown 表格或 CSV/JSON 可互转格式，便于 Multi-Agent 汇总、筛选与后续自动生成教程。
- 每条资料至少保留：标题、作者/账号、平台、链接、发布日期、摘要、主题标签、可信度标签、是否原创、是否适合零基础、可映射章节。
- 对知乎、小红书、公众号等平台内容，需特别增加“经验型/观点型/营销型/教程型”标签，避免把主观感受帖误当方法论。
- 对小红书与短内容平台，优先提取“经验线索”和“真实使用痛点”，不应单独作为高可信技术依据；必要时需用博客、官方文档或长文内容交叉验证。
- 对博客与知乎长文，优先提炼可复用结构，如：如何描述需求、如何让 Cursor 解释代码、如何让 AI 修 bug、何时切换到人工接管。
- 对项目实战资料，应按项目类型归类，例如：静态页面、个人网站、命令行工具、自动化脚本、简单全栈、已有项目维护；便于后续按学习曲线编排。
- 对“代码理解”和“调试”主题，需重点收集真实案例，而不只收集功能介绍；优先关注“如何读取陌生项目”“如何缩小问题范围”“如何验证 AI 生成修复方案”。
- 对“Agent 协作”主题，可同时检索 Cursor 与更广义 AI coding workflow、multi-agent coding、AI pair programming 经验，再筛选与 Cursor 使用场景相关的内容。
- 对“常见误区”主题，应单独建立负面样本集，包括：过度依赖一句话生成整项目、缺少验收标准、上下文过长导致偏航、不会读代码直接盲改、忽略测试与回归验证。
- 建议建立“推荐引用优先级”：官方文档 > 原创深度博客/高质量实战复盘 > 高赞问答/经验帖 > 视频内容转写 > 短内容平台经验线索。
- 若同一内容出现多平台转载，默认保留原作者首发平台，并将其他版本作为传播参考，不作为主来源。
- 对时间敏感内容应记录版本上下文，例如 Cursor 功能、模型能力、界面布局可能变化，需标注资料时间，避免旧教程误导新手。

- 对每个主题可输出“3 层摘要”：一句话结论、要点列表、可教学案例，方便后续直接嵌入 Markdown 教程。
