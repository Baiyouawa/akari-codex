## Goal Description
围绕“小侑零基础计划”输出一份可直接执行的 Markdown 文档结构方案，用于指导 Multi-Agent 系统并行调研、整理、编排并最终汇总为一套完整的 28 天学习教程。该方案需确保最终文档同时具备以下内容：每日教程正文框架、资料来源区块、每周系统课程安排、学习顺序说明、阶段成果目标，并能支持后续多人协作、分模块写作、统一验收与持续补充迭代。

该方案的目标不是直接撰写全部 28 天正文内容，而是先定义最终 Markdown 成品的组织方式、章节层级、模块模板、编写规范、信息汇总逻辑与执行顺序，使不同 Agent 可以按统一标准并行产出内容，最终汇总为一份结构清晰、顺序合理、可学习、可追溯来源、可验收成果的系统化教程文档。

## Acceptance Criteria

- AC-1: 最终方案必须明确给出整份 Markdown 文档的完整层级结构。
  - Positive Tests: 文档中包含总标题、总说明、学习顺序说明、周计划、每日教程模板、资料来源区块、阶段成果目标等一级或二级结构；章节关系清晰，能够直接作为最终成品目录使用。
  - Negative Tests: 仅描述“要写 28 天教程”但未给出实际文档结构；只有零散模块，没有层级关系；缺少某个核心板块。

- AC-2: 方案必须包含“每日教程正文框架”的统一模板设计。
  - Positive Tests: 每日内容至少包含学习目标、前置知识、核心概念、操作步骤或练习、常见问题、当日总结、成果检查等字段。
  - Negative Tests: 每天仅有标题列表，没有正文模板；不同天内容结构完全不统一；无法指导后续 Agent 写作。

- AC-3: 方案必须包含“资料来源区块”的标准结构与收录规则。
  - Positive Tests: 明确资料来源按平台、主题或日期归档；标注来源名称、链接、用途、可信度说明、适用学习阶段等信息。
  - Negative Tests: 仅写“参考资料”；没有来源字段；无法追踪内容出处；不同 Agent 无法统一登记资料。

- AC-4: 方案必须包含“每周系统课程安排”。
  - Positive Tests: 至少划分为第 1 周至第 4 周，并说明每周主题、核心能力、学习重点、练习方向和周成果。
  - Negative Tests: 只有 28 天平铺列表，没有周维度安排；每周安排与每日内容无法对应；没有周目标。

- AC-5: 方案必须包含“学习顺序说明”并体现从零基础到阶段成果的递进逻辑。
  - Positive Tests: 明确先学基础概念，再做基础练习，再进入综合任务，再形成阶段成果；说明为何按该顺序安排。
  - Negative Tests: 内容顺序随机；先出现高阶任务后补基础；没有解释排序依据。

- AC-6: 方案必须包含“阶段成果目标”，且至少覆盖周级或阶段级验收。
  - Positive Tests: 每周结束有具体产出，如术语理解、基础操作完成、小练习作品、综合案例、最终汇总能力等。
  - Negative Tests: 只写“学完即可”；没有中期成果；无法衡量阶段是否完成。

- AC-7: 方案必须适用于 Multi-Agent 并行协作。
  - Positive Tests: 能清楚拆分为资料调研、结构设计、每日内容编写、周计划统筹、质量校验、来源核验等任务；不同模块之间接口明确。
  - Negative Tests: 方案完全依赖单线程串行执行；没有模块边界；并行后无法合并。

- AC-8: 方案必须强调 Markdown 成品的一致性与可汇总性。
  - Positive Tests: 说明标题层级规范、字段命名一致、来源记录格式一致、每日模板一致，便于自动合并或人工整合。
  - Negative Tests: 每部分格式自由发挥；无法拼接成统一文档；后续整理成本过高。

## Path Boundaries

### Upper Bound (Maximum Scope)
最完整的方案应包括：
1. 一份完整的 28 天 Markdown 教程总目录结构。
2. 第 1 周到第 4 周的系统课程安排与主题说明。
3. 每日教程统一正文模板，明确所有固定字段。
4. 每周阶段目标与最终阶段成果目标。
5. 学习顺序设计原则与章节之间的依赖关系说明。
6. 资料来源区块模板，包括平台分类、链接、用途、可信度、适用日期。
7. Multi-Agent 分工建议，包括调研 Agent、写作 Agent、审校 Agent、汇总 Agent。
8. Markdown 编写规范，如标题层级、列表格式、引用格式、代码块或示例块格式。
9. 内容汇总流程，包括资料收集、去重、归类、编排、审核、整合。
10. 面向最终成品的可验收结构，保证后续只需填充内容即可发布。

### Lower Bound (Minimum Scope)
最简可行方案至少应包括：
1. 28 天教程的总结构框架。
2. 每日教程正文模板。
3. 每周课程安排概览。
4. 学习顺序说明。
5. 阶段成果目标。
6. 资料来源区块的最基本字段设计。

即使采用最小方案，也必须保证结构完整、逻辑连续、能支撑后续内容填充与并行协作。

### Allowed Choices
- Can use: Markdown 标题层级设计、周计划表述、每日模板字段、来源引用模板、阶段目标列表、模块化拆分、并行调研任务描述、统一命名规则、平台资料分类方法、学习路径递进说明。
- Cannot use: 只给出泛泛建议而无结构；省略 28 天教程的整体组织方式；缺失资料来源区块；缺失周安排或阶段目标；输出非 Markdown 结构说明；使用无法汇总的随意格式；将执行细节写成程序代码实现。
