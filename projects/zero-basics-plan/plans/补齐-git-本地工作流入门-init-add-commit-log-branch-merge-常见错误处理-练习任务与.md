   - 为每个核心命令统一写作模板：作用、何时用、基本语法、示例、注意事项。

   - 将“概念解释”和“上手操作”分层，降低阅读门槛。

4. 补写核心命令内容
   - `git init`：初始化仓库、`.git` 目录意义、常见误区。
   - `git add`：工作区与暂存区、添加单文件/全部文件、误加场景。
   - `git commit`：提交快照、提交说明规范、首次提交注意事项。
   - `git log`：查看历史、简洁视图、辅助理解提交链。
   - `git branch`：分支概念、本地分支创建/查看/切换关系。
   - `git merge`：合并基本流程、快进合并与冲突感知的入门级说明。

   - 选取新手最常见、最影响上手的错误场景。

6. 设计练习任务与自检方式
   - 基础练习：初始化、提交、查看日志。
   - 中阶练习：创建分支并各自提交。
   - 进阶练习：制造并处理一次简单合并冲突。
   - 给出预期状态，如“应看到几次提交”“应看到哪些分支”“合并后文件应包含什么内容”。

7. 整理参考资料并标注用途

   - 去除重复、低质量、过时内容。

8. 进行交叉校验与并入准备
   - 逐条对照验收标准检查覆盖情况。
   - 实机复跑所有命令示例，确认无明显语法问题。
