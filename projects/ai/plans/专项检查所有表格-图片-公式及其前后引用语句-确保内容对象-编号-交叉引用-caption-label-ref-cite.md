## Goal Description
对 `Project/AI/main.tex` 开展一次“对象保护优先”的专项检查与改写护栏设计，重点覆盖所有表格、图片、公式及其前后引用语句，确保以下内容在深度人类化改写过程中始终保持稳定且可编译一致：

1. 内容对象本体不被修改：包括表格环境、图片环境、公式环境及其内部内容。
2. 结构标识不被破坏：包括编号机制、`caption`、`label`、`\ref`、`\eqref`、`\autoref`、`\cite` 及相关交叉引用链路。
3. 引用语句语义不失真：对象前后用于引出、解释、比较、总结的文字可润色，但不得改变其所指对象、编号关系和结论逻辑。
4. 仅允许对 surrounding prose 做文字层面人类化改写，不触碰对象本体、结构命令和引用命令。
5. 通过多 Agent 协作完成“识别—隔离—审查—回归验证—迭代修正”的闭环，降低 AI 痕迹的同时，避免引入 LaTeX 结构性损坏。

  - Negative Tests: 修改表格数据、替换图片文件名、增删公式项、改动环境类型、调整对象内部换行或对齐导致语义或渲染变化

- AC-2: 所有 `caption`、`label`、`ref`、`eqref`、`autoref`、`cite` 等引用相关命令保持可用且指向正确
  - Positive Tests: 编译后编号连续、交叉引用解析正常、正文提及“表/图/公式/文献”与实际对象一致
  - Negative Tests: `label` 被改名、`\ref{...}` 指向失效、引用语句与对象类型不一致、文献引用键被误改

- AC-3: 对象前后引用语句可润色但不得改变指代关系与事实含义
  - Positive Tests: “如表 2 所示”可改写为“表 2 展示了…”，但仍准确对应原对象与原结论
  - Negative Tests: 将“图 3”误写为“表 3”、将“公式 (5)”误写为“公式 (6)”、将“显示相关性”改成“证明因果性”

- AC-4: 改写范围严格限定于 surrounding prose，不进入对象本体和结构命令内部
  - Positive Tests: 仅修改对象前后自然语言叙述，保留 LaTeX 结构块完整不变
  - Negative Tests: 在 `table`、`figure`、`equation`、`align`、`tabular`、`caption{}`、`\label{}`、`\cite{}` 等内部直接改写

- AC-5: 完成一次全文件级对象清单核对，覆盖所有表格、图片、公式及其引用链
  - Positive Tests: 能产出对象索引，列明对象类型、标签、引用位置、前后文检查状态
  - Negative Tests: 漏检未引用对象、漏检多次引用对象、只检查局部章节而非 `main.tex` 实际纳入内容

- AC-6: 编译与回归检查通过，未因改写造成 LaTeX 错误、警告激增或排版异常
  - Positive Tests: 编译成功；无新增未解析引用；对象编号与目录层级稳定；无明显浮动错乱
  - Negative Tests: 出现 undefined reference、multiply defined labels、citation missing、对象编号重排或引用断裂

- AC-7: 多 Agent 协作分工明确，存在互审与冲突回收机制
  - Positive Tests: 至少包含对象识别、文本改写、引用核对、编译回归、终审汇总等角色分工
  - Negative Tests: 单链路直接覆盖原文、无复核、无冲突仲裁、无修改边界说明

## Path Boundaries

### Upper Bound (Maximum Scope)
最完整方案应包括：

1. 对 `Project/AI/main.tex` 及其 `\input` / `\include` 纳入的所有子文件建立对象资产清单。
2. 自动或半自动识别全部表格、图片、公式环境，以及相关 `caption`、`label`、`\ref`、`\eqref`、`\autoref`、`\cite`。
3. 为每个对象抓取前后固定窗口文本，形成“对象保护区 + 可润色区”映射。
4. 采用多 Agent 集群并行执行：
   - 对象解析 Agent
   - 改写边界控制 Agent
   - 人类化润色 Agent
   - 引用一致性审查 Agent
   - LaTeX 编译回归 Agent
   - 终审裁决 Agent
5. 建立修改白名单与黑名单：

   - 黑名单：对象环境内部、引用命令内部、标签定义、文献键、数学命令
6. 对每一处改写执行三类检查：
   - 语义检查：是否仍指向同一对象

   - 编译检查：是否保持可编译
7. 输出差异报告，按对象维度列出：
   - 未改动对象
   - 仅改动 surrounding prose 的位置
   - 需人工复核的高风险改写点
8. 完成至少一轮迭代修正，对首轮发现的问题进行回收与再验证。

5. 对发现的引用失配或误改进行回滚。

## Implementation Notes
- 代码中不应包含 AC-、Milestone 等计划术语
- 所有表格、图片、公式环境应作为只读保护块处理，优先采用“先识别、后冻结、再改写周边文本”的策略
- 对 `caption{}`、`\label{}`、`\ref{}`、`\eqref{}`、`\autoref{}`、`\cite{}` 的参数内容执行严格保护，不做任何字面改动
- 若引用语句中同时包含多个对象，应逐一核对对象类型、顺序和结论是否保持一致
- 对含数学结论的 surrounding prose 改写时，只能改变表述风格，不能改变比较关系、条件、范围、符号含义和推导结论
- 对含文献引用的句子改写时，不得改变文献归属关系，不得把原本支持性引用改写成对比性或否定性引用
- 若对象前后文本与对象本体高度耦合，宁可少改、不强改，优先保证正确性与稳定性
- 建议保留逐处 diff，便于快速定位是否误触结构命令或对象边界
- 建议以“对象为中心”组织审查，而非仅按章节顺序线性审阅，避免漏检跨段引用
- 对编译通过但语义错指的情况，仍应判定为不合格，必须经过人工或高置信复审
- 若发现原稿已存在坏链路或错误引用，应单独标记，不应在本轮“仅文字层面改写”中擅自结构性修复，除非获得额外授权
