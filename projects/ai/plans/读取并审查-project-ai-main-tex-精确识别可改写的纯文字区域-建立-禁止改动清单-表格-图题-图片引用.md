## Implementation Notes
- 先审查、后改写，避免在未划清边界前直接进入文本润色
- 审查时以 LaTeX 结构完整性为首要目标，宁可少标可改写区域，也不要误放开高风险区域
- 对以下对象默认视为禁止改动：`table`、`tabular`、`figure`、`includegraphics`、数学环境、`\caption{}`、`\label{}`、`\ref{}`、`\eqref{}`、`\cite{}`、章节层级命令、各类 `\begin{...}` / `\end{...}` 环境命令
- 对混合文本采用“最小改写单元”原则：若一句话中夹带引用、公式或交叉引用，仅改动其中不影响结构的自然语言片段
- 若章节标题文本被纳入“章节结构”保护范围，则标题文字本身也不应改写；若后续另有明确授权，需单独重新定义边界后方可处理
- 对自定义宏命令保持保守态度：无法确认其是否承载结构功能时，一律暂列为禁止改动
- 输出结果应使用清晰分类，如“完全禁止”“条件允许”“完全允许”，方便多 Agent 统一执行
- 复核重点应放在容易误伤的区域：图题命令参数、引用命令参数、公式前后说明句、列表项中的命令混排、脚注中的引用与公式
- 审查结论应可被后续执行者直接使用，不需要再次推断哪些内容能改、哪些不能改
- 代码中不应包含 AC-、Milestone 等计划术语
