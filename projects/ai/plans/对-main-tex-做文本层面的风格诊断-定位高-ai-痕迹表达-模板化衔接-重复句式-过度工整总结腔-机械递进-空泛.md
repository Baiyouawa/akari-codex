## Goal Description
对 `Project/AI/main.tex` 开展仅限文本层面的风格诊断与实施规划，系统识别高 AI 痕迹表达，包括但不限于模板化衔接、重复句式、过度工整的总结腔、机械递进、空泛强调、统一长度段落、语义密度分布单一、抽象概括过多、转折与结论模式重复等问题；在不改动表格、图片、公式、章节结构、标题层级、编号体系、引用关系与核心事实含义的前提下，形成可执行的改写策略、审查方法与多轮迭代机制，使正文整体更接近自然写作的人类表达习惯，同时保留原有论证逻辑与学术可读性。

  - Negative Tests: 只给出泛泛评价而无法定位到具体段落；只说“像 AI”但没有可复核的语言证据；遗漏多种高频问题类型

  - Positive Tests: 不修改表格、图片、公式、图注、表注、章节结构、编号、交叉引用与文献键；可对正文句子、段落衔接、措辞密度、语气节奏提出改写建议

  - Negative Tests: 只有零散改词建议，没有系统方法；只关注词汇替换，不处理句法和段落节奏；无法指导后续批量改写

## Path Boundaries

   - 诊断 Agent：逐章扫描并标注问题位置
   - 模式归纳 Agent：汇总高频表达模板与重复句法
   - 改写策略 Agent：为不同问题类型设计替代表达路径
   - 保真审查 Agent：检查语义、术语、逻辑与引用一致性
   - 总控 Agent：合并结果、消解冲突、设定优先级

   - 按章节、节、段落建立索引
   - 标记高风险连接词、套话结尾、整齐三段式递进、过度对称句群等现象
   - 记录重复句式、重复起手方式、重复论证收束方式

5. 形成问题地图与风险排序
   - 区分高风险集中区、中风险普遍区、低风险保留区
   - 判断哪些问题属于局部修辞问题，哪些已影响全文风格一致性
   - 优先处理对“AI 感”贡献最大的段落与开头结尾部分

6. 制定分层改写策略
   - 句级：减少固定连接模板、打散统一句长、削弱空泛强化词、增加自然主语与动作性表达
   - 段级：重排信息密度、避免每段都“提出—解释—总结”过度工整收束、引入更自然的过渡
   - 节级：避免每节都使用同一推进框架，保留必要学术逻辑但弱化机械层级感

7. 设计审查机制与回退规则
   - 审查“去模板化”是否有效
   - 审查语义是否偏移、术语是否变形、论证是否断裂
   - 设定回退条件：一旦改写牺牲准确性、严谨性或引用对应关系，则回滚并重写

8. 输出最终实施蓝图
   - 交付问题分类表、重点段落清单、改写策略矩阵、复核清单
   - 给出建议执行顺序与迭代节奏
   - 确保团队可立刻基于该方案进入正文改写阶段

## Implementation Notes
- 代码中不应包含 AC-、Milestone 等计划术语
- 诊断时优先关注读者最先感知 AI 痕迹的位置，如摘要、引言首段、每节开头、每节结尾、总结合并段
- 不将“学术规范表达”误判为 AI 痕迹，需区分必要正式文体与机械模板化写法
- 对重复问题应同时记录“词汇层重复”和“句法层重复”，避免只做表面替换
- 改写策略应遵循“先去模板，再调节奏，后做细部润色”的顺序
- 应保留专业术语、论证主线与事实指向，不以花哨表达替代准确表达
- 可建立“高风险触发词与结构清单”，例如过多的“首先/其次/最后”“值得注意的是”“综上所述”“由此可见”等，但需结合上下文判断
- 对统一长度段落、整齐排比段、连续同构句群等现象，应从节奏多样性角度提出改写方案，而非简单增删字数
- 审查时需特别检查结论段与实验分析段，二者最容易出现空泛拔高、机械总结和无信息增量的强调句
- 交付物应能被后续改写执行者直接使用，内容应以问题定位、改写原则和复核规则为核心，而非停留在抽象建议层面
